# 记忆数据处理目录

本目录用于存放记忆数据处理相关的过程、服务、处理器和辅助组件。

当前规划中的核心职责包括：

1. 对 `observation_records` 进行清洗、过滤和批处理。
2. 调用宿主授权的大模型能力完成摘要生成、主题生成和记忆类型生成。
3. 将原始事实片段提取为长期记忆片段。
4. 对长期记忆进行去重、合并、评分和归档。
5. 基于多个长期记忆片段生成抽象记忆。
6. 维护处理状态、处理进度、失败信息和审计记录。
7. 保证所有处理结果可追溯到原始观察记录和原文。
8. 后续在需要时纳入向量化处理链路。

建议后续代码按职责拆分：

| 文件或子目录 | 用途 |
| --- | --- |
| `IMemoryProcessingService.cs` | 记忆处理服务入口接口。 |
| `MemoryProcessingService.cs` | 第一版批处理编排服务。 |
| `MemoryProcessingHostedService.cs` | 后台定时处理服务。 |
| `Extractors/` | 候选长期记忆抽取器。 |
| `Classifiers/` | 记忆类型、主题、重要性分类器。 |
| `Summarizers/` | 摘要生成器。 |
| `Mergers/` | 去重和合并处理器。 |
| `Abstractions/` | 抽象记忆生成器。 |
| `Vectorization/` | 向量化处理器。 |
| `Tracing/` | 原文追溯和处理证据链辅助组件。 |
