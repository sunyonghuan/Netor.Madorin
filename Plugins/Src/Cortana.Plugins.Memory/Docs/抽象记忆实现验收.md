# 抽象记忆实现验收

## 范围

对 `MemoryAbstractionService` 与 `FallbackMemoryAbstractionGenerator` 的第一版实现进行验收：

- 验证能否按 topic 聚合 fragments 并生成 memory_abstractions。
- 验证生成结果写入 `memory_abstractions` 表并记录 mutation/event。
- 验证 DI 注入与构建通过。

## 结果

- 已新增接口：`IMemoryAbstractionGenerator`、`IMemoryAbstractionService`。
- 已新增降级生成器：`FallbackMemoryAbstractionGenerator`，基于 fragment.summary 做简单合并，生成 statement 与 summary。
- 已新增处理服务：`MemoryAbstractionService`，实现 `RunAbstractionPass`，按 topic 聚合 fragment 并在满足最小支撑数量时生成抽象。
- 已在 `Startup` 注册 DI。
- 构建通过。

## 限制与后续

- 当前生成器为降级实现，后续应接入宿主授权模型（purpose: memory-abstraction），要求模型输出包含 supporting fragment id 与来源证据。
- 当前抽象生成触发为手动调用 `IMemoryAbstractionService.RunAbstractionPass`；后续应考虑把该功能纳入 `MemoryProcessingHostedService` 的任务队列。
