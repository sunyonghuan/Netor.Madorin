> 目标：采用“扩展内部 WebSocket 控制面协议”的方式，在宿主构建 AI 上下文前主动请求 Memory 插件供应长期记忆包，并由宿主格式化注入 `AIContext.Instructions`。不依赖 AI 主动调用 `memory_supply_context` 工具。

## 当前风险

| 风险 | 说明 | 处理 |
| --- | --- | --- |
| AIContextProvider 拿不到完整当前用户输入 | 需要确认 `InvokingContext` 可提供哪些字段。 | 不足时从 `ChatHistoryDataProvider` / session state 补齐。 |
| 内部 feed 当前可能是单向事件流 | 若现有连接不支持请求/响应，需要补 requestId 映射。 | 保持协议扩展小步实现。 |
| 注入过多污染 prompt | 长期记忆可能与当前任务无关。 | 设置数量、预算、可信度阈值和分组过滤。 |
| 插件慢响应影响主对话 | 记忆系统不能阻塞主链路。 | 宿主设置短超时并静默降级。 |
| agentId 作用域不可靠 | 旧工具默认回退 `default`。 | 控制面请求强制携带 `agentId`，插件缺失即报错。 |
