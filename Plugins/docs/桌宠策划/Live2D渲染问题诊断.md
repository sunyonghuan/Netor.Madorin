## Live2D 渲染问题诊断总结

### 当前状态（2026-05-25）

经过多次尝试，发现"后脑勺"问题**不是简单的坐标或 UV 翻转能解决的**。

### 尝试过的方案

| 方案 | X坐标 | Y坐标 | U坐标 | V坐标 | 结果 |
|------|-------|-------|-------|-------|------|
| 原始 | 正常 | 正常 | 正常 | 翻转 | ❌ 后脑勺，方向正确 |
| 方案1 | 正常 | 翻转 | 正常 | 翻转 | ❌ 倒立 + 后脑勺 |
| 方案2 | 翻转 | 正常 | 正常 | 翻转 | ❌ 左右镜像 + 后脑勺 |
| 方案3 | 翻转 | 翻转 | 正常 | 翻转 | ❌ 倒立 + 后脑勺 |
| 方案4 | 正常 | 正常 | 翻转 | 翻转 | ❌ 纹理完全错乱 |
| 方案5 | 正常 | 正常 | 正常 | 不翻转 | ❌ 纹理错位 |

### 官方 SDK 的处理方式

根据 `CubismEffect.fx`（第 63-64 行）：
```hlsl
Out.uv.x = In.uv.x;           // U 不翻转
Out.uv.y = 1.0f - In.uv.y;    // V 翻转
```

**结论**：官方只翻转 V 坐标，这与我们的原始实现一致。

### 可能的真正原因

"后脑勺"问题可能是：

1. **模型参数问题**：
   - `ParamBodyAngleY` 或类似参数控制模型朝向
   - 默认值可能让模型朝向背面
   - 需要检查模型的默认参数值

2. **Projection Matrix 问题**：
   - 官方使用 `projectMatrix` 进行顶点变换
   - 这个矩阵可能包含镜像或旋转
   - 我们的 CPU 端变换可能缺少某些关键变换

3. **模型本身的设计**：
   - Haru 模型可能默认就是背面朝向
   - 需要通过参数或变换来显示正面

### 下一步建议

1. **检查模型参数**：
   - 读取所有参数的默认值
   - 查看是否有 `AngleY`、`Rotation` 等参数
   - 尝试设置这些参数来改变朝向

2. **实现完整的 MVP 矩阵**：
   - 添加 Constant Buffer
   - 在 Shader 中应用 `projectMatrix`
   - 参考官方 SDK 的矩阵构建方式

3. **对比 Web 预览**：
   - Web 预览显示正面
   - 查看 WebGL 版本如何处理
   - 可能有特殊的参数设置

### 当前代码状态

已恢复到官方的 UV 处理方式：
- U 坐标：不翻转
- V 坐标：翻转（`1.0f - v`）
- 位置坐标：不翻转

这应该与官方 SDK 的处理一致，但仍然显示后脑勺。

### 需要进一步调查

问题的根源不在 UV 或坐标翻转，而可能在于：
- 模型的初始参数设置
- Projection Matrix 的构建方式
- 或者 Haru 模型本身的特殊设计

**建议暂停坐标翻转的尝试，转而研究模型参数和 MVP 矩阵的实现。**
