## 🔍 技术原理

### 问题根因
Live2D Cubism 使用标准数学坐标系（Y+ 向上），但在映射到屏幕空间时需要考虑 D3D11 的视口变换。不翻转 Y 轴会导致：
1. 模型上下颠倒
2. 背面 drawable（后脑勺、背部）显示在前面
3. 正面 drawable（脸部）被遮挡

### 解决方案
在 CPU 端顶点变换时对 Y 坐标取反：
```
screenY = -((modelY - centerY) * scaleY)
```

这确保了：
- 模型头部（正 Y）→ 屏幕上方（正 Y）
- 模型脚部（负 Y）→ 屏幕下方（负 Y）
- 正确的 drawable 渲染顺序

### 与官方 SDK 的对比
官方 Cubism D3D11 SDK 通过 MVP 矩阵在 shader 中处理坐标变换：
```hlsl
Out.position = mul(float4(In.pos, 0.0f, 1.0f), projectMatrix);
```

我们的实现在 CPU 端完成变换，更简单但需要手动处理 Y 轴翻转。
