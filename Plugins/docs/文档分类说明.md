# Plugins/docs 文档分类说明

> 整理时间：2026年5月14日

## 目录约定

### 未来版本策划

路径：`Plugins/docs/未来版本策划/`

用于存放：

- 尚未实现的插件方案；
- 仍处设计阶段的插件框架方案；
- 未完成的执行计划；
- 待实测、待实操、待原型验证的报告或实验方案；
- 后续版本才考虑落地的插件能力。

完成实现、验证或发布后，应移动到 `Plugins/docs/已完成功能规划/`。

### 已完成功能规划

路径：`Plugins/docs/已完成功能规划/`

用于存放：

- 已有对应插件实现的方案文档；
- 已落地的主线架构设计；
- 已完成的执行步骤；
- 已完成或已成为当前实现依据的契约文档。

### 参考文档

路径：`Plugins/docs/参考文档/`

用于存放：

- 当前开发者参考文档；
- 插件使用说明；
- 通道说明；
- 不属于“未来策划”或“已完成功能规划”的长期参考资料。

## 后续使用规则

1. 新插件方案先放入 `Plugins/docs/未来版本策划/`。
2. 插件实现完成并验证后，移动到 `Plugins/docs/已完成功能规划/`。
3. 插件开发者教程、协议说明、长期参考资料放入 `Plugins/docs/参考文档/`。
4. 一个目录中如果既有完成内容又有未完成实验，应拆分：完成部分进入已完成，实验部分进入未来策划。
5. 状态为“待实测”“待实操”“待原型实现”“0%”的文档默认属于未来策划。
