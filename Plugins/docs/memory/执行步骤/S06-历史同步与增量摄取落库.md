# S06 - 历史同步与增量摄取落库

> 目标：在插件侧建立最小数据库（SQLite），完成“历史回放 + 增量订阅”的一致摄取，形成 ObservationRecords 基础数据层。

## 范围
- 协议：采用《协议总览》v1.1.0 新增的 Conversation Feed Replay（replay/batch/completed）。
- 插件：建库/入库/去重（按消息 Id），落表 `observation_records`（最小列集）。
- 宿主：在 conversation-feed 上实现 `replay` 控制与批量导出 `conversation.export.batch`。
- 增量：继续消费 `conversation.*` 事件，补全实时摄取。

## 实施任务
- 宿主
  - [ ] `WebSocketServerService` 支持 `type=replay`，按 `sinceTimestamp` 查询 ChatMessages，分批回放。
  - [ ] 批次帧与完成帧序列化（源生成上下文）。
  - [ ] 可配置 batchSize（默认 500，范围 100–2000）。
  - [ ] 观测：导出总数与用时日志。
- 插件（Memory）
  - [ ] 初始化 `memory.db`（位于 `PluginSettings.DataDirectory`）。
  - [ ] 建表 `observation_records(id TEXT PRIMARY KEY, sessionId TEXT, role TEXT, content TEXT, createdTimestamp INTEGER, modelName TEXT)`。
  - [ ] 启动时请求 `replay(since=0)`，批量插入（忽略主键冲突）。
  - [ ] 订阅 `conversation.*` 事件，按最小映射补充写入。
  - [ ] 记录导入总数与增量写入速率。

## 非功能
- AOT：所有导出/回放 DTO 采用源生成上下文。
- 幂等：以 `id` 为主键去重（重复批次/重连均安全）。
- 可用性：replay 失败不阻塞主链路，插件可退化为“仅增量摄取”。

## 验收清单（DoD）
- [ ] 插件首次启动完成回放并输出“导入 N 条”日志。
- [ ] 宿主日志输出回放总数与用时；大数据量下分批发送。
- [ ] 增量事件到达后 `observation_records` 行数持续增长。
- [ ] 重启插件后重复回放不产生重复行（主键去重生效）。
