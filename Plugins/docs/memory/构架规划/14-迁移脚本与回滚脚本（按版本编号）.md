# 14 - 迁移脚本与回滚脚本（按版本编号）

> 状态：第一版
> 对应验收项：Step 9.2

## 目标

提供可执行的数据库迁移与回滚脚本模板，支撑 N-2 / N-1 / N 兼容发布。

## 迁移策略

- 使用 Expand -> Migrate -> Contract 三阶段。
- Expand 与 Migrate 必须可回滚。
- Contract 必须在双写稳定后单独发版。

## 版本编号规则

- V1：历史基线版本。
- V2：引入 ObservationRecords 与版本字段。
- V3：移除旧兼容字段（Contract 阶段）。

## V1 -> V2 迁移脚本（Expand + Migrate）

```sql
BEGIN TRANSACTION;

-- 1) 新增 ObservationRecords
CREATE TABLE IF NOT EXISTS ObservationRecords (
	Id TEXT PRIMARY KEY,
	AgentId TEXT NOT NULL,
	WorkspaceId TEXT NULL,
	SessionId TEXT NOT NULL,
	TurnId TEXT NOT NULL,
	MessageId TEXT NULL,
	EventType TEXT NOT NULL,
	Role TEXT NULL,
	Content TEXT NULL,
	AttachmentsJson TEXT NULL,
	Timestamp TEXT NOT NULL,
	TraceId TEXT NOT NULL,
	SourceFactsJson TEXT NOT NULL,
	SchemaVersion INTEGER NOT NULL DEFAULT 2,
	RecordVersion INTEGER NOT NULL DEFAULT 1,
	CreatedAt TEXT NOT NULL
);

-- 2) LongMemories 新增版本化与状态字段（按现网情况调整）
ALTER TABLE LongMemories ADD COLUMN WorkspaceId TEXT NULL;
ALTER TABLE LongMemories ADD COLUMN ClarityLevel TEXT NOT NULL DEFAULT 'blurred';
ALTER TABLE LongMemories ADD COLUMN ConfirmationState TEXT NOT NULL DEFAULT 'pending';
ALTER TABLE LongMemories ADD COLUMN LifecycleState TEXT NOT NULL DEFAULT 'candidate';
ALTER TABLE LongMemories ADD COLUMN SchemaVersion INTEGER NOT NULL DEFAULT 2;
ALTER TABLE LongMemories ADD COLUMN RecordVersion INTEGER NOT NULL DEFAULT 1;

-- 3) RuleMemories 新增版本字段
ALTER TABLE RuleMemories ADD COLUMN SchemaVersion INTEGER NOT NULL DEFAULT 2;
ALTER TABLE RuleMemories ADD COLUMN RecordVersion INTEGER NOT NULL DEFAULT 1;

-- 4) 迁移记录
INSERT INTO SchemaMigrations (Version, AppliedAt, Description, IsRollbackSafe)
VALUES (2, datetime('now'), 'Expand/Migrate to V2', 1);

COMMIT;
```

## V2 -> V1 回滚脚本（仅回滚安全窗口）

```sql
BEGIN TRANSACTION;

-- 1) 保护性检查：仅允许回滚到标记安全版本
-- 由应用层先验证 SchemaMigrations.IsRollbackSafe = 1

-- 2) 删除 V2 新表（需确认无关键数据未备份）
DROP TABLE IF EXISTS ObservationRecords;

-- 3) 记录回滚
INSERT INTO SchemaMigrations (Version, AppliedAt, Description, IsRollbackSafe)
VALUES (1, datetime('now'), 'Rollback to V1', 1);

COMMIT;
```

## V2 -> V3 迁移脚本（Contract）

```sql
BEGIN TRANSACTION;

-- 示例：移除旧兼容列前，先创建新表并搬迁（SQLite 不支持直接 DROP COLUMN）
CREATE TABLE IF NOT EXISTS LongMemories_v3 AS
SELECT
	Id, AgentId, WorkspaceId, MemoryType, Topic, Title, Summary, Detail,
	KeywordsJson, TagsJson, EntitiesJson,
	EvidenceObservationIdsJson, EvidenceSessionIdsJson, EvidenceTurnIdsJson,
	Importance, Confidence, SalienceScore, RetentionScore, DecayRate,
	AccessCount, ReinforcementCount, ContradictionCount,
	ClarityLevel, ConfirmationState, LifecycleState,
	LastAccessedAt, LastReinforcedAt, ExpiresAt,
	SchemaVersion, RecordVersion, CompatibilityTagsJson,
	CreatedAt, UpdatedAt
FROM LongMemories;

DROP TABLE LongMemories;
ALTER TABLE LongMemories_v3 RENAME TO LongMemories;

INSERT INTO SchemaMigrations (Version, AppliedAt, Description, IsRollbackSafe)
VALUES (3, datetime('now'), 'Contract to V3', 0);

COMMIT;
```

## 执行清单

- 先备份数据库文件与索引。
- 先执行 Expand，再执行 Migrate。
- 验证双写窗口稳定后再执行 Contract。
- Contract 发布后禁止自动回滚到 V1。
