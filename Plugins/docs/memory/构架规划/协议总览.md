# 协议总览（Conversation/Memory）

> 版本：1.1.0（在 1.0.0 基础上新增 Conversation Feed 的历史回放 Replay 控制）

## 分层一览
- 聊天通道：`ws://localhost:{port}/ws/`（旧）
- 内部事件：`ws://localhost:{port}/internal/conversation-feed/`（事实流）
  - v1.0.0：connected/subscribe/subscribed + event(conversation.*)
  - v1.1.0：新增 replay 控制（request → batch* → completed）

端口下发策略：
- 聊天通道：顶层 `wsPort`（主字段）。
- 事实流通道：`extensions.conversationFeedPort`（主字段）。
- 兼容保留（Deprecated）：`extensions.chatWsEndpoint`、`extensions.conversationFeedEndpoint`。
- 控制通道：后续补（能力申请/召回）

## Conversation Feed v1.0.0（已落地）
- 握手：`connected`（服务端→客户端）
- 订阅：`subscribe`（客户端→服务端）→ `subscribed`（服务端→客户端）
- 事件：`{ type:"event", topic:"conversation", eventType, payload }`
- 事件类型：
  - `conversation.turn.started`
  - `conversation.user.message`
  - `conversation.assistant.delta`
  - `conversation.turn.completed`

## Conversation Feed v1.1.0：历史回放（Replay）
> 目标：插件在冷启动时先回放历史消息，再进入增量订阅，完成“历史+实时”的一致摄取。

### 控制消息
- 请求（客户端→服务端）：
```json
{ "type":"replay", "sinceTimestamp": 0, "batchSize": 500 }
```
  - `sinceTimestamp`：Unix 毫秒，缺省=0 表示全量。
  - `batchSize`：每批条目数，默认 500。

- 批次（服务端→客户端）：
```json
{
  "type":"event",
  "topic":"conversation",
  "eventType":"conversation.export.batch",
  "payload":{
    "batchId":"uuid",
    "hasMore": true,
    "items":[
      { "id":"...", "sessionId":"...", "role":"user|assistant|tool", "content":"...", "createdTimestamp": 1710000000000, "modelName":"gpt-x" }
    ]
  }
}
```

- 完成（服务端→客户端）：
```json
{
  "type":"event",
  "topic":"conversation",
  "eventType":"conversation.export.completed",
  "payload": { "total": 12345 }
}
```

### 兼容性
- v1.1.0 新增 `replay` / `conversation.export.*` 可选能力；v1.0.0 客户端将忽略未知帧。
- 事件 `payload` 强类型定义见源生成 JsonContext。
