# 09 - 认知式记忆模型

> 状态：认知模型第一版定稿

## 目标

把“像人的记忆”抽象成系统内可落地的认知结构，补齐表意识、潜意识、清晰度、确认状态、联想、强化、衰退、遗忘、摘要升级这些机制。

## 基本判断

人的记忆不是一个平铺数据库，而是一个带层次、带强弱、带情境、带联想、带不确定性的动态网络。

它至少有以下特征：

- 有些记忆在表意识里，随时能说出来
- 有些记忆在潜意识里，平时不出现，遇到线索才会被唤起
- 有些记忆非常清楚，有些只有模糊印象
- 有些记忆是确定事实，有些只是怀疑、猜测或待确认
- 有些记忆带很强情感权重，因此更难遗忘
- 有些记忆本身不重要，但能作为联想入口带出别的记忆
- 人不会把全部记忆同时摆到脑子里，而是按需要激活少量记忆

系统如果想接近这种能力，就不能只建“存储结构”，还要建“记忆状态结构”和“激活结构”。

## 系统必须表达的 6 条维度

### 1. 内容维度

记忆本身属于什么内容。

- 情感
- 工作
- 技能
- 情景
- 关系
- 目标
- 偏好
- 经验

### 2. 意识维度

记忆当前处于表意识、前意识还是潜意识。

### 3. 清晰度维度

记忆当前是清晰、一般、模糊，还是只剩轮廓。

### 4. 确认维度

记忆当前是已确认、待确认、被质疑，还是只是暂存假设。

### 5. 激活维度

记忆当前是否正在被使用、刚被唤起，还是长期沉寂。

### 6. 联想维度

一个记忆是否容易通过人、事、地点、时间、情绪、目标、因果关系去联想到别的记忆。

## 意识层模型

### 1. 表意识层

当前轮真正进入上下文的记忆层。

特点：

- 数量必须很少
- 相关性必须高
- 默认可以直接对宿主注入
- 更接近“我现在脑子里正在想什么”

适合进入这一层的内容：

- 当前目标直接相关的高价值记忆
- 刚刚被再次触发的偏好或经验
- 当前问题必须依赖的稳定抽象结论

### 2. 前意识层

平时不在上下文里，但给一点线索就能很快想起来。

特点：

- 不直接注入
- 检索成本低
- 通常作为第一次联想扩展池

### 3. 潜意识层

长期沉在底层，不主动出现，但可能通过联想链被唤醒。

特点：

- 平时不进入上下文
- 只能通过检索或图扩展唤起
- 很多旧经验、旧场景、旧关系会沉在这一层

### 4. 反思层

不是原始经历，而是从大量经历中形成的抽象认知层。

例如：

- 用户偏好画像
- 常见失败模式
- 稳定工作方式
- 项目长期理解

这一层更像“人对自己经历做出的总结”。

## 清晰度模型

记忆不只分有或没有，还要分“清楚到什么程度”。

建议统一成 4 档：

- `vivid`：非常清楚，细节和来源都比较完整
- `clear`：主体清楚，但细节不一定全
- `blurred`：只有大致印象，需要借助其他线索补全
- `faint`：只剩轮廓，更多是一个联想入口

清晰度的意义：

- 决定是否适合直接注入上下文
- 决定是否需要先做二次确认
- 决定是否只能当作联想跳板，不能直接当事实使用

## 确认状态模型

不是所有记忆都能直接当成真相使用。

建议统一成 5 档：

- `confirmed`：已有足够证据，可直接使用
- `likely`：大概率成立，但仍建议保留不确定提示
- `pending`：暂存记忆，需要二次确认
- `contested`：已有反证或冲突信息
- `hypothesis`：只是从片段推出来的猜测

这一维度非常关键，因为它决定了系统能不能把某条记忆直接注入给智能体，还是只能作为“待核验线索”。

## 联想模型

人不是靠全量扫描来回忆，而是靠线索触发联想。

系统里建议把联想定义成：

`线索 -> 种子记忆 -> 邻接记忆 -> 过滤 -> 召回包`

### 线索来源

能触发联想的线索至少包括：

- 人：用户、智能体、相关角色
- 事：任务、问题、工具调用、失败事件
- 地：工作区、项目、模块、文件
- 时：时间段、最近会话、某个阶段
- 情：情绪色彩、紧急程度、挫败感、满意度
- 目标：当前要完成什么
- 因果：为什么发生、由什么导致

### 联想关系

联想不是只有相似，还应包括：

- 相似联想
- 因果联想
- 时间前后联想
- 目标关联联想
- 角色关联联想
- 地点或项目关联联想
- 情绪关联联想
- 经验到结论的抽象联想

### 联想约束

为了避免污染，联想不能无边界扩散。

建议约束：

- 默认只允许 1 到 2 跳联想
- 第二跳必须比第一跳更严格过滤
- 清晰度过低且确认状态不高的记忆，不允许直接进入上下文
- 潜意识层被唤醒后，必须重新排序，不能一律带出

## 抽象升级模型

系统不应该一开始就把每段对话直接升格为“长期认知”。

建议升级路径：

`ObservationRecord -> MemoryFragment -> EpisodeSummary -> MemoryAbstraction -> StableProfile`

### 1. ObservationRecord 到 MemoryFragment

从事实记录抽取出一个值得保留的经历片段。

适用条件：

- 有明确情景
- 有明确行为或结论
- 对未来可能有价值

### 2. MemoryFragment 到 EpisodeSummary

把一次完整经历压成一个更短的事件摘要。

适用条件：

- 同一任务或同一主题下片段过多
- 需要降低注入成本
- 原始细节已不需要全部保留在高层

### 3. EpisodeSummary 到 MemoryAbstraction

从多次经历中提炼出稳定结论。

例如：

- 用户偏好某种输出形式
- 某类问题经常在同一个环节出错
- 某个项目需要先做什么再做什么

触发条件建议至少满足其一：

- 多次重复出现
- 被多条片段共同支持
- 被再次召回且证明有用
- 被人工确认

### 4. MemoryAbstraction 到 StableProfile

把稳定抽象再汇聚成更高层画像。

例如：

- 用户画像
- 项目画像
- 智能体工作风格画像
- 长期目标画像

这一层不应太多，只保留最稳定、最核心、最慢变化的认知。

## 缩略摘要模型

缩略摘要不是简单截短文本，而是按层级压缩信息密度。

建议做 4 层摘要：

### 1. 片段摘要

把一个 fragment 压成 1 到 3 句，保留：

- 发生了什么
- 为什么重要
- 结果是什么

### 2. 事件摘要

把同一任务下多个 fragment 压成一次 episode summary。

保留：

- 背景
- 关键过程
- 最终结论
- 是否有失败或风险

### 3. 主题摘要

把多个 episode 的共同模式压成 abstraction summary。

保留：

- 规律
- 适用范围
- 置信度
- 反例或边界条件

### 4. 画像摘要

把长期稳定 abstraction 汇总成 profile summary。

例如：

- 用户在沟通中的长期偏好
- 智能体在工作中的稳定策略
- 某项目最应优先遵守的高阶规则

越往上层，文本越短，但要求来源链越强。

## 强化、衰退与遗忘规则

### 1. 强化规则

记忆在以下情况下应被强化：

- 被再次提及
- 被召回后确实帮助了当前任务
- 被新的片段再次支持
- 被人工确认
- 与当前目标高度相关并连续命中

强化后的直接结果：

- 提升激活度
- 提升清晰度
- 提升保留倾向
- 降低短期被遗忘概率

### 2. 衰退规则

记忆在以下情况下应自然衰退：

- 长期未访问
- 长期未被支持
- 当前目标体系中不再相关
- 被更高质量记忆覆盖

衰退后的直接结果：

- 从表意识降到前意识
- 从前意识降到潜意识
- 清晰度下降
- 联想权重下降

### 3. 遗忘规则

遗忘不等于立刻删除，而是分层处理：

- 先从表意识退出
- 再从前意识退到潜意识
- 再从潜意识进入归档或逻辑遗忘
- 只有合规或人工明确要求时才做物理删除

### 4. 反证规则

如果新证据与旧记忆冲突：

- 不直接覆盖旧记忆
- 标记为 contested 或 hypothesis
- 降低其可注入优先级
- 建立反证链
- 必要时要求二次确认

## 二次确认机制

系统要允许“这个记忆我似乎记得，但还不够确定”。

以下情况建议进入二次确认：

- 清晰度低于 `clear`
- 确认状态不是 `confirmed`
- 关键结论只来自单一片段
- 存在反证或冲突关系
- 需要把它作为高影响决策依据

二次确认通过后，记忆才允许升级到更高层或进入表意识注入。

## 第一版建议公式

第一版不需要把公式做得很复杂，但最好从一开始就统一成同一套分数语言。

### 1. 激活分数

建议把“当前有多容易被想起”统一成：

`activationScore = recency + goalMatch + recentUse + emotionWeight + associationDensity - contradictionPenalty`

它主要决定记忆更接近表意识、前意识还是潜意识。

### 2. 保留分数

建议把“这条记忆值不值得长期留着”统一成：

`retentionScore = importance + confidence + reinforcementCount + abstractionValue - decayPenalty - duplicatePenalty`

它主要决定记忆是继续保留、归档还是进入遗忘。

### 3. 清晰度分数

建议把“记得有多清楚”统一成：

`clarityScore = sourceIntegrity + detailLevel + recentRecall + repeatedSupport - agePenalty`

它主要决定记忆能不能直接注入，还是只能作为待确认线索。

### 4. 联想权重

建议把“这个记忆能不能顺带带出别的记忆”统一成：

`associationWeight = entityOverlap + goalOverlap + temporalProximity + causalStrength + emotionalResonance`

它主要决定图扩展时先走哪条边。

### 5. 注入优先级

建议把“最终是否进入上下文”统一成：

`injectionPriority = activationScore + retentionScore + relevanceToCurrentTask - uncertaintyPenalty - noisePenalty`

这一步一定要把不确定和噪声惩罚算进去，否则长期记忆很快就会污染当前上下文。

## 上下文注入原则

智能体不能被强制灌入太多记忆，否则长期记忆就会变成噪声源。

因此应遵循：

- 默认只把表意识层的少量高价值记忆注入上下文
- 前意识和潜意识层只在检索命中后参与
- 抽象记忆优先于原始片段
- 原始片段只保留少量作为证据
- 待确认、模糊、被质疑的记忆不直接作为事实注入

## 第一版建议的认知流程

```text
事实进入
	-> 提取经历片段
	-> 判断是否值得保留
	-> 进入候选记忆
	-> 随时间衰退或被再次强化
	-> 在多次支持下升级为抽象记忆
	-> 通过线索触发联想召回
	-> 只把少量高价值记忆送入表意识上下文
```

## 结论

长期记忆系统最终不应表现为“存了很多内容”，而应表现为：

- 知道什么该记住
- 知道什么该淡化
- 知道什么还不确定
- 知道什么时候应该联想
- 知道联想出来的内容里什么能直接使用，什么必须先确认

这才更接近人的记忆，而不是日志系统。
