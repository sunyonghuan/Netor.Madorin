# 04 - 插件接入与初始化扩展

> 状态：已迁移到 S08 单端点 PluginBus 初始化字段  
> 当前主线字段：`pluginBusEndpoint`、`pluginBusPort`、`pluginBusProtocol`、`pluginBusVersion`  
> 说明：旧 `conversationFeedEndpoint`、`conversationFeedPort`、`modelCapabilityEndpoint` 只保留为历史文档概念，不再作为 Memory 插件内部通信入口。

## 目标

为新标准插件定义初始化扩展槽位和内部事件端点下发方式。

## 当前现状

当前插件 init 参数仅包含：

- `dataDirectory`
- `workspaceDirectory`
- `pluginDirectory`
- `wsPort`

这只能满足旧聊天 WebSocket 接入需求，无法表达新的内部事件流端点。

## 当前已落地槽位

当前插件 init 配置已新增：

- `extensions`：字符串字典扩展槽位，类型语义为 `Dictionary<string, string>`

当前结构：

```json
{
	"dataDirectory": "xxx",
	"workspaceDirectory": "xxx",
	"pluginDirectory": "xxx",
	"wsPort": 52841,
	"extensions": {
		"pluginBusEndpoint": "ws://localhost:65321/internal",
		"pluginBusPort": "65321",
		"pluginBusPath": "/internal",
		"pluginBusProtocol": "madorin.plugin-bus",
		"pluginBusVersion": "1.0.0"
	}
}
```

## 设计原则

- `wsPort` 当前仍复用为 PluginBus 端口兼容字段，真正语义以 `extensions.pluginBusPort` 为准。
- 新标准插件不再依赖旧 `/ws/` 或 `/internal/conversation-feed/` 推导内部事件地址。
- 内部事件流、memory supply 与 model capability 均通过 `pluginBusEndpoint` 传入。
- Native / Process SDK 都会把完整扩展字段保留为 `PluginSettings.Extensions`
- 已有常用扩展字段可以继续通过兼容属性读取，但新增扩展不再修改 `PluginSettings` 属性集合
- 扩展值统一为字符串；如果插件需要传递复杂 JSON，由宿主写入 JSON 字符串，插件侧自行反序列化

### 字段去重决策（当前）

- 顶层字段：
	- `wsPort` 暂时保留并填入 `pluginBusPort`，用于兼容 Native 初始化结构。
- 扩展字段（extensions）：
	- `pluginBusEndpoint` 为 Memory 插件连接主字段。
	- `pluginBusPort` 为端口主字段。
	- `pluginBusPath` 固定为 `/internal`。
	- `pluginBusProtocol` 固定为 `madorin.plugin-bus`。
	- `pluginBusVersion` 固定为 `1.0.0`。
- 插件消费建议：
	- 优先读取 `pluginBusEndpoint`。
	- 缺失时读取 `pluginBusPort` 并组装 `ws://localhost:{port}/internal`。
	- 不再读取 `conversationFeedEndpoint` 或 `ConversationFeedEndpoint` 兼容属性。

## 当前宿主下发链路

- `ExternalProcessPluginHostBase.InitializePluginAsync(...)` 负责组装 `extensions`
- `NativePluginInitConfig` 增加 `extensions` 字符串字典槽位并走源生成序列化
- `MadorinWsEndpoints` 统一产出 `pluginBusEndpoint`。

## 当前插件运行时可见字段

- `PluginSettings.Extensions`
- `PluginSettings.Extensions["pluginBusEndpoint"]`
- `PluginSettings.Extensions["pluginBusPort"]`
- `PluginSettings.Extensions["pluginBusProtocol"]`
- `PluginSettings.Extensions["pluginBusVersion"]`

## 当前兼容策略

- 旧字段 `wsPort` 继续保留，但当前填入的是 PluginBus 端口。
- Memory 插件只使用 `pluginBusEndpoint` / `pluginBusPort`。
- 若扩展字段缺失，Memory 插件会记录配置缺失并进入退避重试，不再回退到 `conversationFeedEndpoint`。
