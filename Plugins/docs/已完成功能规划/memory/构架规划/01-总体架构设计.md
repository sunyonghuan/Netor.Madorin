# 01 - 总体架构设计

> 状态：宿主事实流第一版已落地

## 目标

建立宿主分发事实流、插件独立建设长期记忆的总体架构。

## 目标架构总览

```text
┌──────────────────────────────────────────────┐

│ WebSocketConversationFeedRelayService        │
│ WebSocketServerService                       │

└──────────────┬───────────────────────────────┘
			   │ 内部事件 WebSocket

│ 私有数据库 / 私有索引 / 私有向量库           │
└──────────────────────────────────────────────┘
```

## 分层设计

### 1. 工作记忆层

- 当前轮对话上下文
- 当前轮输入输出内容
- 当前轮工具调用过程
- 当前轮临时中间状态

### 2. 会话事实层

- 原始聊天消息
- 会话 ID / Turn ID
- 附件
- 角色
- 智能体名称
- 智能体ID
- 时间戳
- 模型信息

### 3. 长期记忆层

- 记忆片段
- 关系连接
- 抽象记忆
- 偏好
- 失败经验
- 可检索索引

### 4. 规则记忆层

- 当前工作区生效
- 高优先级
- 人工维护
- 不参与自动成长
- 直接进入 AI 指令上下文

## 核心职责划分

### 主程序职责

- 统一接收所有对话输入
- 统一驱动 AI 推理
- 统一发布对话事实事件
- 统一提供内部 feed 只读订阅出口
- 不负责长期记忆内部存储和向量实现

### 宿主模型路由前置原则

- 插件申请宿主 LLM 能力之前，必须先在宿主设置层完成“用途级模型路由”
- 第一批至少要能配置：主对话模型、对话压缩摘要模型、记忆加工模型
- 后续如果其他插件也需要申请模型能力，仍然复用同一套宿主设置，不允许各插件私自维护自己的模型选择
- 插件侧只表达“我要申请哪一类能力”，不直接决定最终使用哪个 Provider / Model
- 记忆插件的 LLM 使用属于宿主能力借用，不属于插件自带模型体系

## 当前已落地宿主链路

- `AiChatHostedService.SendMessageAsync(...)` 负责发布 turn started / user message / assistant delta / turn completed
- `Events.cs` 作为宿主内部对话事实总线的统一事件定义入口
- `WebSocketConversationFeedRelayService` 负责把 Conversation 事件转成内部 feed 的 `event` 消息
- `WebSocketServerService` 继续承载旧聊天 `ws://localhost:{port}/ws/`，同时新增 `ws://localhost:{port}/internal/conversation-feed/`

### 记忆插件职责

- 接收对话事实流
- 进行记忆抽取、分类和聚合
- 构建私有记忆数据库
- 建立关系网络和检索索引
- 产出召回结果供宿主使用
