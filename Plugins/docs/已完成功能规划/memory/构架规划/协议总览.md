# 协议总览（Conversation/Memory）

> 版本：2.0.0（S08 单端点 PluginBus 主线）  
> 当前主线：`ws://localhost:{pluginBusPort}/internal`，协议 `cortana.plugin-bus`，版本 `1.0.0`  
> 说明：旧 Conversation Feed v1.0/v1.1 设计已被 PluginBus 取代，旧内容不再作为实现依据。

## 分层一览

- 插件总线：`ws://localhost:{pluginBusPort}/internal`
  - 连接控制：`connected`、`subscribe`、`subscribed`、`error`
  - 心跳：`ping` / `pong`
  - 对话事实：`topic = conversation`
  - 历史回放：`conversation.history.replay` → `conversation.history.batch*` → `conversation.history.completed`
  - 长期记忆供应：`memory.context.supply.request` → `memory.context.supply.response/error`
  - 模型能力：`model.capability.request` → `model.capability.response`

端口下发策略：

- 主字段：`extensions.pluginBusEndpoint`
- 端口字段：`extensions.pluginBusPort`
- 协议字段：`extensions.pluginBusProtocol = cortana.plugin-bus`
- 版本字段：`extensions.pluginBusVersion = 1.0.0`
- 顶层 `wsPort` 暂时复用 PluginBus 端口，仅作为 Native 初始化兼容字段。

## PluginBus v1.0.0（当前已落地）

基础 envelope 字段：

| 字段 | 说明 |
|---|---|
| `type` | 消息类型，例如 `event`、`request`、`response`、`error`、`ping`、`pong` |
| `protocol` | 固定为 `cortana.plugin-bus` |
| `version` | 固定为 `1.0.0` |
| `topic` | `conversation`、`memory`、`model`、`plugin` |
| `op` | 操作名 |
| `requestId` | 请求响应关联 ID |
| `payload` | 业务载荷 |

已使用 topic：

- `conversation`
- `memory`
- `model`
- `plugin`

## Conversation 历史回放
> 目标：插件在冷启动时先回放历史消息，再进入增量订阅，完成“历史+实时”的一致摄取。

### 控制消息
- 请求（客户端→服务端）：
```json
{
  "type": "request",
  "protocol": "cortana.plugin-bus",
  "version": "1.0.0",
  "topic": "conversation",
  "op": "conversation.history.replay",
  "requestId": "uuid",
  "payload": {
    "sinceTimestamp": 0,
    "batchSize": 500
  }
}
```
  - `sinceTimestamp`：Unix 毫秒，缺省=0 表示全量。
  - `batchSize`：每批条目数，默认 500。

- 批次（服务端→客户端）：
```json
{
  "type": "response",
  "protocol": "cortana.plugin-bus",
  "version": "1.0.0",
  "topic": "conversation",
  "op": "conversation.history.batch",
  "requestId": "uuid",
  "payload": {
    "batchId": "uuid",
    "hasMore": true,
    "items": [
      { "id": "...", "sessionId": "...", "role": "user|assistant|tool", "content": "...", "createdTimestamp": 1710000000000, "modelName": "gpt-x" }
    ]
  }
}
```

- 完成（服务端→客户端）：
```json
{
  "type": "response",
  "protocol": "cortana.plugin-bus",
  "version": "1.0.0",
  "topic": "conversation",
  "op": "conversation.history.completed",
  "requestId": "uuid",
  "payload": { "total": 12345 }
}
```

约束：最后一个 `hasMore=false` 的 batch 发送后，宿主必须立即发送 `conversation.history.completed`。

### 兼容性
- v1.1.0 新增 `replay` / `conversation.export.*` 可选能力；v1.0.0 客户端将忽略未知帧。
- 事件 `payload` 强类型定义见源生成 JsonContext。

## 长期记忆供应

当前实现使用：

- `memory.context.supply.request`
- `memory.context.supply.response`
- `memory.context.supply.error`

说明：当前代码仍保留 Memory supply 顶层 DTO 生成格式，双方可通信；迁移为 `payload` envelope 是 S08-25 的剩余标准化任务。

## 模型能力

当前实现使用同一条 PluginBus 长连接：

- `model.capability.request`
- `model.capability.response`

Memory 插件通过 `requestId` 匹配模型能力响应。

## 废弃项

以下旧项不再作为当前实现依据：

- `/ws/` 作为 Memory 插件内部通道
- `/internal/conversation-feed/`
- `conversation-feed` 协议名
- `conversation.export.batch`
- `conversation.export.completed`
- `memory.supply.request`
- `memory.supply.package`
- `memory.supply.error`
