# 10 - 记忆系统实现策划（程序与算法）

> 状态：第一版落盘

## 目标

把“规则记忆”和“长期记忆”从概念层转为可编码、可调度、可治理的程序系统。

规则记忆负责强约束与高优先级注入，长期记忆负责跨会话成长与经验沉淀。

两者在检索阶段汇合，由统一编排器在 Token 预算内输出可注入上下文。

## 程序分层

- Rule Memory Layer：工作区级、人工维护、强约束、直接注入。
- Long-term Memory Layer：智能体级、自动提取、跨会话、检索召回。
- Memory Orchestrator：统一执行写入流水线与读取流水线。
- Governance Layer：冲突治理、衰减、归并、可审计。

## 总体流程

Conversation Event -> Candidate Extraction -> Scoring -> Merge -> Persist -> Retrieval -> Context Injection

## AOT 风险评估

- 可安全实现：规则匹配、打分排序、SQLite 持久化、确定性裁决。
- 高风险项：运行时反射、多态动态序列化、Expression 动态编译。
- 约束策略：强类型 DTO + 源生成 JsonSerializerContext。
- 插件边界：长期记忆插件只读宿主事件流，不回写宿主事实。

## 写入流水线（算法）

1. 事件接入：接收用户消息、工具调用、工具结果、助手输出、反馈信号。
2. 候选抽取：规则抽取优先，模型抽取补充，统一输出结构化候选。
3. 记忆打分：importance、frequency、recency、confidence 多因子加权。
4. 去重归并：指纹哈希 + 相似度合并，避免重复污染。
5. 持久化：按规则记忆/长期记忆分仓落盘并记录证据链。

建议综合分：

score = 0.35 * importance + 0.25 * frequency + 0.20 * recency + 0.20 * confidence

其中 recency 建议使用指数衰减：

recency = exp(-lambda * ageDays)

## 读取流水线（算法）

1. 意图解析：识别任务域、风险域、约束域。
2. 规则优先检索：先取规则记忆并硬注入。
3. 长期记忆检索：主题检索 + 语义检索 + 时间衰减排序。
4. 冲突裁决：规则优先；长期记忆内部按新鲜度与置信度裁决。
5. 预算裁剪：按 Token 上限输出 Recall Package。

## 治理策略

- 低分候选不入库，降低噪声。
- 冲突条目进入待验证状态，禁止直接高权重召回。
- 周期归并任务把碎片记忆压缩为稳定认知节点。
- 所有记忆均保留来源与版本，支持可追溯审计。

## 落地顺序

1. 先完成规则记忆注入改造（最短见效路径）。
2. 再接入长期记忆写入流水线与分层存储。
3. 最后上线冲突治理、衰减与周期归并。

此顺序可在最小风险下快速验证“可控 + 可成长”双目标。
