# 15 - 回滚后读写链路验证用例

> 状态：第一版
> 对应验收项：Step 5.4

## 目标

验证数据库从 V2 回滚到 V1 后，核心读写链路仍可用且不破坏历史数据。

## 前置条件

- 已完成 V1 -> V2 迁移并有测试数据。
- SchemaMigrations 中 V2 记录 IsRollbackSafe = 1。
- 已备份回滚前数据库文件。

## 用例清单

### Case 1 回滚后规则记忆读取

- 步骤：执行 V2 -> V1 回滚，启动宿主，读取 RuleMemories。
- 断言：规则记忆可被正常加载并注入。

### Case 2 回滚后主链路写入

- 步骤：发送一轮对话，触发宿主消息落盘。
- 断言：主对话写入成功，无阻塞或异常中断。

### Case 3 回滚后长期记忆降级读取

- 步骤：请求 Recall，触发长期记忆读取。
- 断言：系统走兼容降级路径，返回可用 RecallPackage。

## 通过标准

- 所有 Case 无致命错误。
- 主对话链路可读可写。
- Recall 链路可返回可消费结构。

## 证据留档

- 执行日志（含 traceId）。
- 回滚前后表结构快照。
- 用例执行截图或命令输出摘要。
