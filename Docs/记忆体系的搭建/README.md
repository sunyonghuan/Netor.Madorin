## 背景

当前系统已经具备统一 AI 对话入口、聊天消息落库、工作区级文件记忆、插件隔离和 WebSocket 通信能力。

但这些能力仍然不足以支撑一套真正面向智能体长期成长的记忆体系。

现有文件记忆本质上是工作区规则注入，不是长期记忆网络。

现有聊天历史压缩主要解决上下文窗口问题，也不是长期记忆网络。

## 核心决策

| 项目 | 决策 | 理由 |
|---|---|---|
| 长期记忆引擎 | 独立插件 / 子进程实现 | 与宿主隔离，便于演进向量索引和图谱能力 |
| 会话事实来源 | 宿主内部对话事件总线 | 避免插件直接读取主程序数据库 |
| 插件接入方式 | 内部事件 WebSocket 订阅 | 低污染、跨语言、便于隔离 |
| 规则记忆 | 保留并重新定位 | 它属于工作区规则层，不属于长期成长记忆 |
| 旧插件兼容 | 不做兼容 | 平台仍新，直接推进新标准更干净 |
| 主程序数据库 | 不对插件开放 | 降低污染和安全边界风险 |

## 结论

本规划不再把聊天数据库视作记忆系统本体，而是把它视作事实源之一。

真正的长期记忆体系将作为独立插件运行，通过宿主内部事件流接收对话事实，自行建立记忆库、索引和关系网络。
