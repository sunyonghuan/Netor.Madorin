# 02 - 对话事件总线设计

> 状态：第一版已落地

## 目标

在统一 AI 消息循环基础上建立宿主内部对话事实总线。

## 设计原则

- 事件只描述事实，不描述用途
- 事件必须可追溯到会话和轮次
- 事件必须支持流式输出
- 事件必须覆盖完成、取消和失败三类收尾状态
- 事件必须与现有语音 / UI 生命周期事件分离

## 当前落地事件集合

| 事件名 | 说明 |
|---|---|
| `conversation.turn.started` | 一轮对话开始 |
| `conversation.user.message` | 本轮用户消息进入 AI 主流程 |
| `conversation.assistant.delta` | AI 流式输出片段 |
| `conversation.turn.completed` | 一轮结束，状态为 succeeded / cancelled / failed |

## 复用的既有事件

- `session.created` 继续承担会话创建事实
- `session.title.updated` 继续承担会话标题更新事实
- `ai.started` / `ai.completed` 继续承担 UI 生命周期信号，不并入 Conversation 事实事件族

## 当前统一字段

- `occurredAt`
- `sessionId`
- `turnId`
- `userMessageId`
- `assistantMessageId`
- `providerId`
- `providerName`
- `agentId`
- `agentName`
- `modelId`
- `modelName`
- `traceId`

## 当前挂点

- 在 [Src/Netor.Cortana.AI/AiChatHostedService.cs](Src/Netor.Cortana.AI/AiChatHostedService.cs) 的 `SendMessageAsync` 中生成 `turnId` 和 `traceId`
- Session 准备完成后发布 `conversation.turn.started`
- 用户消息进入主流程后发布 `conversation.user.message`
- `RunStreamingAsync` 每个 assistant 文本增量发布 `conversation.assistant.delta`
- 成功、取消、异常三条收尾路径统一发布 `conversation.turn.completed`

## 当前未纳入字段

- `sourceChannel`：当前 `IAiChatEngine.SendMessageAsync` 不携带输入来源，宿主此时无法可靠区分 UI / Voice / WebSocket。该字段延后到输入层改造时补齐。
- `workspaceDirectory`：这不是每条对话事实都必需的主键字段，后续若插件握手需要可通过 init 参数提供。

## 结论

新的对话事件总线已经以第一版最小闭环方式落地到 EventHub：

- 负责承载 turn 级事实，不承载 UI 信号语义
- 统一了 `sessionId / turnId / traceId` 三个追踪主轴
- 为下一步内部事件 WebSocket 提供了稳定的宿主内事件源
