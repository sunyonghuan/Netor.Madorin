# 11 - 记忆系统细化设计（接口与数据结构）

> 状态：生产版基线（含向后兼容）

## 设计目标

把第 10 篇策划转为可直接指导编码和发布的规格文档。

本设计同时满足：AOT 安全、生产可用、灰度发布、可回滚、向后兼容。

## 模块边界

- 宿主：事件发布、规则记忆注入、用途级模型路由、RecallPackage 消费。
- 记忆插件：长期记忆抽取、打分、检索、归并、治理。
- 边界约束：插件仅读宿主事实流，不可反向改写宿主事实库。

## 生产可用非功能要求

- 可用性：记忆子系统故障不阻塞主对话，降级到规则记忆 + 聊天历史压缩。
- 性能：Recall 构建超时即降级，避免拖慢主链路。
- 一致性：写入采用幂等 eventId + 去重指纹，防止重复入库。
- 可观测：全链路 traceId，记录命中率、冲突率、降级率、回滚率。
- 可恢复：支持启动自检、索引重建、坏数据隔离、在线修复。

## 向后兼容策略

### 协议版本策略

- 内部事件协议与 Recall 协议采用 major.minor 版本。
- minor 仅允许追加可选字段，旧端默认忽略未知字段。
- major 变更必须带适配层，不允许破坏性替换。

### 数据版本策略

- 每张核心表保留 SchemaVersion 列。
- 每条核心记录保留 RecordVersion 列。
- 读取走 VersionedReader，写入走 CurrentWriter。

### 行为兼容策略

- 新策略默认关闭，使用 FeatureFlag 灰度。
- 保留 LegacyRecallPipeline 作为回退路径。
- 宿主对旧插件至少保持 N-2 兼容。

## 枚举与状态约束

LifecycleState
- candidate, active, dormant, reinforced, contradicted, archived, forgotten, deleted

ConfirmationState
- confirmed, likely, pending, contested, hypothesis

ClarityLevel
- vivid, clear, blurred, faint

约束规则：

- 新记忆默认 candidate。
- contested 与 hypothesis 不允许直接高优先级注入。
- forgotten 不参与正常检索，仅用于审计与恢复。

## 迁移与回滚

- 采用 Expand -> Migrate -> Contract 三阶段。
- Expand：加新列加新表，不删旧列。
- Migrate：开启双写，读优先新结构，失败回落旧结构。
- Contract：双写稳定后单独版本移除旧字段。
- 回滚仅允许回退到 IsRollbackSafe 版本。

## 版本兼容映射（N-2 / N-1 / N）

- Host(N) + Plugin(N-2)：走 BuildRecallPackageCompatible，字段降级到旧协议子集。
- Host(N) + Plugin(N-1)：启用新字段读取，保留旧字段回退。
- Host(N) + Plugin(N)：启用完整字段与新治理策略。
- Host(N-1) + Plugin(N)：插件输出兼容字段集，不发送破坏性字段。

## 运行时时序（写入）

1. 宿主发布 ConversationEvent。
2. ObservationStore 落盘 ObservationRecord。
3. CandidateExtractor 从 ObservationRecord 抽取候选。
4. ScoringEngine 计算 salience 与 retention。
5. MergeEngine 去重归并，LongMemoryStore 落盘并写证据链。

## 运行时时序（读取）

1. 解析请求场景与意图。
2. RuleMemoryProvider 拉取高优先级规则。
3. LongMemoryStore 检索长期记忆并做衰减排序。
4. ConflictResolver 处理冲突与状态门控。
5. BudgetReducer 生成 RecallPackage 并注入宿主。

## 关键伪代码

OnConversationEvent(evt):
- obs = SaveObservation(evt)
- candidates = Extract(obs)
- scored = Score(candidates)
- merged = Merge(scored)
- SaveLongMemories(merged)

BuildRecallPackage(req):
- rules = GetRules(req.workspaceId, req.scenario)
- memories = QueryLong(req.agentId, req.query)
- gated = FilterByState(memories)
- resolved = ResolveConflicts(rules, gated)
- return ReduceByBudget(resolved, req.budget)

## AOT 实现约束

- 所有序列化对象使用强类型 DTO。
- Json 使用源生成上下文，避免动态反射。
- 插件协议字段固定化，避免 object 任意扩展。
- 禁止运行时表达式编译作为核心路径。

## 生产发布门禁

- 兼容测试：N-2、N-1、N 互通。
- 迁移测试：从线上主流版本直升当前版本。
- 回滚测试：升级后回滚至上一个稳定版本并保持可读可写。
- 故障演练：关闭记忆插件后主对话仍可用。

## SLO 建议（可验收）

- Recall 构建 P95 <= 150ms，P99 <= 300ms。
- 事件写入成功率 >= 99.95%。
- 记忆子系统降级触发率 <= 0.5%。
- 冲突未决积压超过 24h 的数量 <= 100。
