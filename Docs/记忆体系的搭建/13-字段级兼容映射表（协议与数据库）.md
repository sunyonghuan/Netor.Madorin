# 13 - 字段级兼容映射表（协议与数据库）

> 状态：第一版
> 对应验收项：Step 9.1

## 目标

定义 Host 与 MemoryPlugin 在 N-2 / N-1 / N 版本组合下的字段级兼容规则。

## 版本语义

- N：当前版本。
- N-1：上一个稳定版本。
- N-2：上上个稳定版本（最低兼容边界）。

## 适用范围

- 内部事件协议（Conversation Feed）。
- RecallPackage 交付协议。
- SQLite 核心表结构与字段。

## 通用兼容规则

- minor 只允许新增可选字段。
- major 变更必须提供兼容适配层。
- 未识别字段必须忽略，不允许抛错中断主链路。

## 协议字段映射（RecallPackage）

| 字段 | N | N-1 | N-2 | 兼容策略 |
|---|---|---|---|---|
| RequestId | required | required | required | 强一致透传 |
| AgentId | required | required | required | 强一致透传 |
| WorkspaceId | optional | optional | missing | N/N-1 读；N-2 默认 null |
| QueryIntent | required | optional | missing | 旧端缺失时默认 `unknown` |
| HitFragments | required | required | optional | 缺失时降级为 LongMemories |
| SupportingFragments | required | optional | missing | 缺失时置空数组 |
| SuppressedMemories | required | optional | missing | 缺失时置空数组 |
| ProducerVersion | required | optional | missing | 缺失按连接协商版本推断 |

## 协议字段映射（CapabilityRequest / CapabilityGrant）

| 字段 | N | N-1 | N-2 | 兼容策略 |
|---|---|---|---|---|
| Purpose | required | required | required | 枚举值未知时回退 `memory-processing` |
| MaxTokens | required | optional | missing | 缺失时用宿主默认配额 |
| TimeoutMs | required | optional | missing | 缺失时用宿主默认超时 |
| RateLimitPolicy | required | optional | missing | 缺失时使用默认限流策略 |
| ExpiresAt | required | optional | missing | 缺失时按短期租约处理 |

## 数据库字段映射（LongMemories）

| 字段 | N | N-1 | N-2 | 兼容策略 |
|---|---|---|---|---|
| WorkspaceId | optional | optional | missing | 缺失时按 agent 级检索 |
| SalienceScore | required | required | optional | 缺失时由 Score 代算 |
| RetentionScore | required | optional | missing | 缺失时按 Confidence 估算 |
| ClarityLevel | required | optional | missing | 缺失默认 `blurred` |
| ConfirmationState | required | optional | missing | 缺失默认 `pending` |
| LifecycleState | required | required | optional | 缺失默认 `candidate` |
| SchemaVersion | required | optional | missing | 缺失视为 1 |
| RecordVersion | required | optional | missing | 缺失视为 1 |

## 数据库字段映射（ObservationRecords / RuleMemories）

| 表.字段 | N | N-1 | N-2 | 兼容策略 |
|---|---|---|---|---|
| ObservationRecords.SourceFactsJson | required | optional | missing | 缺失时从 PayloadJson 回填 |
| ObservationRecords.TraceId | required | required | optional | 缺失时由 turn 级 trace 补齐 |
| RuleMemories.SchemaVersion | required | optional | missing | 缺失视为 1 |
| RuleMemories.RecordVersion | required | optional | missing | 缺失视为 1 |

## 读写矩阵

- Writer(N) -> Reader(N-2)：写新读旧，按字段降级导出。
- Writer(N-1) -> Reader(N)：读路径支持旧字段补全。
- Writer(N) -> Reader(N)：全字段直读，保留兼容分支。

## 实现约束

- 兼容适配器必须纯函数化，禁止隐式副作用。
- 缺省值策略必须可审计并可配置。
- 任意降级路径都不得覆盖原始证据链字段。

## 验收检查

- 每个映射条目必须有对应单测。
- N-2/N-1/N 组合至少各通过 1 组端到端用例。
- 兼容失败必须可观测，记录字段名与版本对。
