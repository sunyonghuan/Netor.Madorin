# 视觉模型与多模态能力规划 : 0%

> 创建时间：2026年5月13日  
> 调整时间：2026年5月14日  
> 范围：图片预览、AI 返回资源显示、图片生成、视频生成、厂商生成端点兼容。

## 一、当前结论

- [√] 模型输入/输出能力标记已具备，是否支持 `Image`、`Video` 由模型设置决定。
- [√] 图片作为用户附件发送给视觉模型识别已验证可用。
- [√] 图片、音频、视频、文件等资源可保存到聊天资源目录，并写入 `ChatMessageAssets`。
- [×] 用户发送图片后当前只显示链接，缺少图片预览。
- [×] AI 返回图片、视频、音频、压缩包、普通文件后，需要统一显示为可点击资源。
- [×] 图片/视频生成需要走独立生成入口，不走普通会话自然语言判断。

## 二、核心方案

### 1. 显式生成入口

- [×] 在聊天输入区域增加模式选项卡：`对话`、`图片`、`视频`。
- [×] `对话`：默认模式，继续走普通会话；图片附件识别也走普通会话。
- [×] `图片`：直接进入图片生成流程，用户输入作为图片提示词。
- [×] `视频`：直接进入视频生成流程，用户输入作为视频提示词。
- [×] 不做自然语言意图识别，不额外调用 LLM 判断是否生成图片/视频。

### 2. 厂商生成端点兼容

- [×] 模型能力只负责判断是否允许输入/输出图片或视频。
- [×] 厂商驱动负责决定图片/视频生成调用哪个端点。
- [×] OpenAI 图片生成可适配 `/images/generations`。
- [×] OpenAI 兼容聊天协议不等于兼容图片生成协议，不能默认所有厂商都支持 `/images/generations`。
- [×] 其他厂商由对应驱动或配置声明图片/视频生成端点。

### 3. 资源显示规则

- [×] 用户发送图片：显示 Markdown 图片预览，不只显示链接。
- [×] AI 返回图片：保存到本地后显示 Markdown 图片预览。
- [×] AI 返回视频：保存到本地后显示 Markdown 文件链接，不内置播放。
- [×] AI 返回音频：保存到本地后显示 Markdown 文件链接，暂不内置播放。
- [×] AI 返回压缩包/文档/普通文件：保存到本地后显示 Markdown 文件链接。
- [×] 非图片资源链接文字必须包含文件名和大小。
- [×] 点击本地文件链接时使用系统默认程序打开。
- [×] 文件不存在时显示明确提示。

推荐显示格式：

- 图片：`![图片名](file:///本地路径)`
- 文件：`[文件名（大小）](file:///本地路径)`

## 三、实施步骤

### Step 1 修复图片预览显示 : 0%

- [×] 用户发送图片后生成 Markdown 图片预览。
- [×] 历史加载时恢复用户图片预览。
- [×] AI 返回图片保存后生成 Markdown 图片预览。
- [×] 图片加载失败时显示占位或错误提示。

### Step 2 统一文件资源显示 : 0%

- [×] AI 返回非图片资源后先保存到聊天资源目录。
- [×] 生成 Markdown 文件链接，显示文件名和大小。
- [×] 支持视频、音频、压缩包、文档、普通文件统一链接显示。
- [×] 点击链接打开本地文件。
- [×] 用户发送的非图片附件也使用同一套链接显示规则。

### Step 3 接入图片生成 : 0%

- [×] 增加 `图片` 模式选项卡。
- [×] 校验当前模型是否包含 `OutputCapabilities.Image`。
- [×] 调用当前厂商的图片生成端点。
- [×] 将 URL、base64、二进制返回统一保存为本地图片资源。
- [×] 写入 `ChatMessageAssets`。
- [×] 在当前会话立即显示生成图片，并支持历史恢复。

### Step 4 接入视频生成 : 0%

- [×] 增加 `视频` 模式选项卡。
- [×] 校验当前模型是否包含 `OutputCapabilities.Video`。
- [×] 调用当前厂商的视频生成端点。
- [×] 支持异步任务状态：排队、生成中、成功、失败、取消。
- [×] 生成完成后保存视频资源并写入 `ChatMessageAssets`。
- [×] 在会话中显示视频文件链接，点击后用系统默认播放器打开。

## 六、验收标准

- 用户发送图片后能在聊天窗口直接预览。
- AI 返回图片能立即预览，并能从历史恢复。
- AI 返回视频、音频、压缩包和普通文件时显示文件名、大小和可点击链接。
- 点击本地资源链接可打开对应文件。
- `对话`、`图片`、`视频` 三种模式路由清晰，不依赖 LLM 意图判断。
- 图片生成和视频生成由厂商驱动适配对应端点。
