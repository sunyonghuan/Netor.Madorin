# 执行计划(模型能力协议简化与插件元数据读取) : 90%

## 背景

当前 Memory 插件通过 `HostModelCapabilityClient` 调用主程序的大模型能力，但存在两个设计问题：

1. **协议职责过重**：插件侧请求/响应包含了过多宿主模型能力细节，例如协议名、操作名、输出结构、预算、模型相关响应字段等。Memory 插件实际只需要“输入 -> 输出”，不应承担不同模型协议、消息结构或厂商适配复杂度。
2. **插件元数据硬编码**：`HostModelCapabilityClient` 当前硬编码 `PluginId = "memory_engine"`，协议版本也硬编码。运行时应优先从插件元数据获取，例如发布/调试输出目录下的 `plugin.json`；调试阶段可保留最小 fallback，但不应作为主路径。

## 目标

- 将宿主大模型能力调用设计为简单的输入输出网关。
- 主程序负责授权、厂商、模型、协议、工具禁用、选项和错误映射。
- 插件只提交任务输入和指令，只接收最终文本或错误。
- 插件运行时从 `plugin.json` 或宿主下发配置读取插件 ID、协议版本等元数据，避免硬编码。

## 设计结论

主程序应定位为“授权模型网关”，插件应定位为“业务调用方”。插件只提交业务输入和指令，只接收最终输出；厂商、模型、协议、工具、token、usage、日志等复杂逻辑均由主程序负责。
