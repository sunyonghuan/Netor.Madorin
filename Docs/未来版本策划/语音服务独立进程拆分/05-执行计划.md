# 05 - 执行计划（语音服务插件化拆分）: 0%

## 总体顺序

本计划不再采用固定 `Cortana.Stt.exe` / `Cortana.Tts.exe` 专用子进程方案，而是改为：

1. 扩展插件能力发现；
2. 先拆 TTS；
3. 再拆 STT；
4. 最后拆 KWS；
5. 验证稳定后从主程序发布包移除语音模型。

执行原则：

- 每一步都要可编译、可运行、可回退；
- 旧实现先保留，确认稳定后再删除；
- 不新增语音专用协议；
- 控制命令走现有 Process 插件 `invoke`；
- 流式事件走现有宿主 WebSocket / EventHub 映射。
