# 04 - 设置项设计

## 设置项目标

语音能力插件化后，设置项不再只是“是否启动固定 STT/TTS 子进程”，而是需要支持：

- 启用/关闭 KWS、STT、TTS 三类能力；
- 从已安装插件中选择当前使用的 KWS/STT/TTS 插件；
- 插件缺失、插件崩溃、插件更新后的状态提示；
- 后续支持多个模型/多个供应商并存。

## 新增设置项

建议在 `SystemSettingsService.SeedDefaults()` 中新增“语音服务”分组，放在现有“语音唤醒”“语音识别”“语音合成”参数分组之前。

| Key | 分组 | 显示名 | 类型 | 默认值 | 说明 |
|---|---|---|---|---|---|
| `Voice.Kws.Enabled` | 语音服务 | 启用关键词唤醒 | bool | true | 是否启用 KWS 插件 |
| `Voice.Kws.PluginId` | 语音服务 | 关键词唤醒插件 | string | 空 | 当前选中的 KWS 插件，空表示自动选择 |
| `Voice.Stt.Enabled` | 语音服务 | 启用语音识别 | bool | true | 是否启用 STT 插件 |
| `Voice.Stt.PluginId` | 语音服务 | 语音识别插件 | string | 空 | 当前选中的 STT 插件，空表示自动选择 |
| `Voice.Tts.Enabled` | 语音服务 | 启用语音合成 | bool | true | 是否启用 TTS 插件 |
| `Voice.Tts.PluginId` | 语音服务 | 语音合成插件 | string | 空 | 当前选中的 TTS 插件，空表示自动选择 |

## 自动选择规则

当 `PluginId` 为空时：

1. `VoiceCapabilityRegistry` 查找已加载且启用的对应能力插件；
2. 优先选择标记为默认的插件；
3. 没有默认插件时选择第一个可用插件；
4. 没有可用插件时，该能力标记为不可用，但主程序继续运行。

## 联动关系

| 场景 | KWS | STT | TTS | 行为 |
|---|---|---|---|---|
| 全部开启 | ✅ | ✅ | ✅ | 完整语音对话 |
| 关闭 KWS | ❌ | ✅ | ✅ | 不监听唤醒词，可保留未来手动语音入口 |
| 关闭 STT | ✅ | ❌ | ✅ | 唤醒后可播问候语，但无法识别用户语音 |
| 关闭 TTS | ✅ | ✅ | ❌ | 可语音输入并获得文字回复，但不播报 |
| 全部关闭 | ❌ | ❌ | ❌ | 纯文字模式，主程序不拉起语音插件 |
| 插件缺失 | ⚠️ | ⚠️ | ⚠️ | 对应能力不可用，其他能力不受影响 |

## 运行时生效策略

### 第一阶段：重启生效

为了降低复杂度，初期可以要求以下设置重启生效：

- 启用/关闭 KWS/STT/TTS；
- 切换 KWS/STT/TTS 插件；
- 安装或更新语音插件。

### 第二阶段：热切换

插件系统稳定后支持热切换：

1. 停止当前能力插件；
2. 调用旧插件 `destroy`；
3. 加载新插件并 `init`；
4. 更新 `VoiceCapabilityRegistry`；
5. 编排器切换到新插件。

## 与旧设置项的关系

现有“语音唤醒”“语音识别”“语音合成”分组中的参数继续保留，例如：

- 唤醒词阈值；
- 识别静音规则；
- 识别超时；
- TTS 语速；
- 欢迎语。

这些参数由主程序统一管理，再通过对应插件的 `configure` 工具传入插件。

如果某插件不支持某个参数：

- 插件可忽略；
- 或在状态中标记“不支持”；
- 主程序不应因为单个参数不支持而阻止插件运行。

## 插件状态展示

建议后续在设置页或插件页展示：

| 字段 | 说明 |
|---|---|
| 插件名称 | 当前选中的语音插件 |
| 插件版本 | 用于判断是否需要更新 |
| 模型名称 | 插件上报的模型名 |
| 模型路径 | 插件内部模型目录，可选展示 |
| 运行状态 | 未启动 / 启动中 / 运行中 / 错误 / 已禁用 |
| 最近错误 | 插件启动失败、模型缺失、设备占用等 |

## 配置下发

主程序在以下时机调用对应插件的 `configure` 工具：

1. 插件 `init` 成功后；
2. 用户修改语音参数后；
3. 插件热切换完成后；
4. 模型或欢迎语变更后。

配置载荷保持 JSON 字符串，由具体插件按需解析。
