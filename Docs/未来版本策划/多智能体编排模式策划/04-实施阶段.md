### 1.2 Coordinator instructions 内容

```text
你是一个多智能体协调者。用户在本轮对话中 @ 提及了 {N} 个子智能体：

{每个子智能体的 Name / Description 列表}

请按以下规则工作：

1. 先制定一个简短计划（不要超过 5 个步骤）。
2. 根据任务需要调用一个或多个子智能体工具。调用工具时：
   - 必须按工具签名传入 `attachmentPaths` 和 `attachmentDescriptions`（如果用户提供了附件）
   - 不要把不相关的附件传给所有子智能体
3. 不要把子智能体的输出直接拼接，必须做冲突检查和汇总。
4. 最终输出包括：
   - 最终回答（清晰、结构化）
   - 简短的"参与智能体"列表（在末尾，作为过渡方案；阶段 2A 起由宿主接管）
```

**目标**：把"何时使用 Coordinator 模式 / 何时使用 SDK Workflow"的判断从 `AIAgentFactory` 抽出来，放到独立的 `IAgentOrchestrator`。**保持现有 Chat 行为不变，但为 Track A 后续阶段（HandoffChat）和 Track B（Workflow Mode 区分判定）准备契约入口**。

```csharp
public interface IAgentOrchestrator
{
    /// <summary>
    /// 仅服务 Chat 模式（None / ToolDelegation / HandoffChat）。
    /// 根据请求构建一个 AIAgent，可以是普通 ChatClientAgent，也可以是 HandoffWorkflow.AsAIAgent 包装。

}
```

## 阶段 3A：Chat 内 HandoffChat（客服分流）

**目标**：在 Chat 模式内引入 SDK `HandoffWorkflowBuilder` 实现客服式分流，**所有事件仍走 `conversation` topic**，用户感知不到换人。

### 3A.1 关键技术路径

利用 `Workflow.AsAIAgent()`（[01-SDK源码研究笔记.md](./01-SDK源码研究笔记.md) §6），把 HandoffWorkflow 包成 AIAgent：

```csharp
// AgentOrchestrator.BuildAgentAsync(...)
return request.Mode switch
{
    AgentOrchestrationMode.None => BuildSingleAgent(request),
    AgentOrchestrationMode.ToolDelegation => BuildCoordinatorAgent(request),
    AgentOrchestrationMode.HandoffChat => BuildHandoffChatAgent(request),  // ← 阶段 3A 新增
    _ => throw new ArgumentOutOfRangeException(...)
};
```

### 3A.2 BuildHandoffChatAgent 实现

```csharp
private AIAgent BuildHandoffChatAgent(AgentOrchestrationRequest request)
{
    var triageAgent = _factory.Build(LoadAgent(request.TriageAgentId), ...);

#pragma warning disable MAAIW001
    var builder = AgentWorkflowBuilder
        .CreateHandoffBuilderWith(triageAgent);

    foreach (var target in request.HandoffTargets)
    {
        var targetAgent = _factory.BuildSubAgent(LoadAgent(target.AgentId), ...);
        builder = builder.WithHandoffs(triageAgent, targetAgent);
    }

    var workflow = builder
        .EnableReturnToPrevious()  // 后续轮次直接路由到上次专家
        .Build();
#pragma warning restore MAAIW001

    return workflow.AsAIAgent(
        id: $"handoffchat-{request.SessionId}",
        name: "HandoffChat",
        description: "Customer-service triage and routing");
}
```

### 3A.3 触发场景

| 触发 | 描述 |
| --- | --- |
| 用户 @ 一个 triage agent + 多个候选专家 | 阶段 3A 默认场景 |
| Agent 自身配置 `IsTriageAgent = true`（待评估） | 阶段 5+ |
| 用户在系统设置选择"客服模式默认 triage" | 阶段 5+ |

### 3A.4 历史保存与事件

- HandoffChat **不切换 topic**：`conversation.turn.started` / `conversation.assistant.delta` / `conversation.turn.completed` 全部沿用现有协议
- 一轮对话内有几个 Agent 切换，对外呈现仍是"该 session 助手回复了一条消息"
- 工具调用链（含 `handoff_to_<agent_id>`）正常入 ChatMessages，与 ToolDelegation 路径一致
- Memory 插件继续按现有协议入库，**无需任何变更**

### 3A.5 验收标准

- ✅ Triage 智能体能识别问题域并触发 Handoff
- ✅ `EnableReturnToPrevious()` 让后续轮次直接路由到上次专家
- ✅ Handoff 链路长度受 `HandoffMaxChainLength` 限制
- ✅ 取消、超时、单 Agent 失败正常处理
- ✅ 现有 None / ToolDelegation 模式行为完全不变
- ✅ Memory 插件无需变更

预计改动：2-3 个新文件（HandoffChat 适配代码 + 测试），1 个文件修改（`AgentOrchestrator.cs` 扩展分支）。

```csharp
public async Task<string> StartTaskAsync(WorkflowTaskRequest request, CancellationToken ct)
{
    var taskId = ...;
    _taskRepo.Insert(...);
    _publisher.Publish(WorkflowEvents.OnWorkflowTaskStarted, ...);

    var workflow = BuildGroupChatWorkflow(request);

    using var taskCts = CancellationTokenSource.CreateLinkedTokenSource(ct);
    taskCts.CancelAfter(_options.PerTaskTimeout);

    try
    {
        await using var run = await InProcessExecution.RunStreamingAsync(
            workflow, request.InitialInput, taskCts.Token);

        await foreach (var evt in run.WatchStreamAsync().WithCancellation(taskCts.Token))
        {
            switch (evt)
            {
                case AgentRunUpdateEvent update:
                    // 累积 Step 输出，但不发流式 PluginBus 事件（决策 3-A）
                    AccumulateStepUpdate(taskId, update);
                    break;

                case ExecutorCompletedEvent completed:
                    // Step 完成，落 OrchestrationStep 表 + 发 workflow.step.completed
                    PersistStep(taskId, completed);
                    _publisher.Publish(WorkflowEvents.OnWorkflowStepCompleted, ...);
                    break;

                case WorkflowOutputEvent output:
                    // Workflow 完成
                    var finalReport = ExtractFinalReport(output);
                    _taskRepo.UpdateCompleted(taskId, finalReport, WorkflowTaskStatus.Completed);
                    _publisher.Publish(WorkflowEvents.OnWorkflowTaskCompleted, ...);
                    break;
            }
        }
    }
    catch (OperationCanceledException) when (taskCts.IsCancellationRequested)
    {
        _taskRepo.UpdateCompleted(taskId, null, WorkflowTaskStatus.Cancelled);
        _publisher.Publish(WorkflowEvents.OnWorkflowTaskCancelled, ...);
    }
    catch (Exception ex)
    {
        _taskRepo.UpdateCompleted(taskId, null, WorkflowTaskStatus.Failed, ex.Message);
        _publisher.Publish(WorkflowEvents.OnWorkflowTaskFailed, ...);
    }

    return taskId;
}
```

```csharp
public sealed class MemoryWorkflowEventHandler(
    PluginSettings settings,
    ILogger<MemoryWorkflowEventHandler> logger,
    IMemoryStore store)
{
    public bool TryHandle(JsonElement root, string? op)
    {
        // 仅处理 workflow.task.completed（决策 2-B）
        var eventType = root.TryGetProperty("eventType", out var et) ? et.GetString() : null;
        if (!string.Equals(eventType, "workflow.task.completed", StringComparison.Ordinal))
        {
            return false;
        }

        if (!root.TryGetProperty("payload", out var payload) || payload.ValueKind != JsonValueKind.Object)
        {
            return true;
        }

        var status = GetString(payload, "status");
        if (!string.Equals(status, "succeeded", StringComparison.OrdinalIgnoreCase))
        {
            return true;  // 失败/取消不入库
        }

        var record = new ObservationRecord
        {
            Id = GetString(payload, "taskId") ?? Guid.NewGuid().ToString("N"),
            AgentId = GetString(payload, "managerAgentId"),
            AgentName = GetString(payload, "managerAgentName"),
            WorkspaceId = ResolveWorkspaceId(payload),
            SessionId = GetString(payload, "sourceSessionId") ?? string.Empty,  // 可空，记录触发来源
            TurnId = null,
            MessageId = GetString(payload, "taskId"),
            EventType = "workflow.task.completed",
            Role = "assistant",
            Content = GetString(payload, "finalReport"),
            AttachmentsJson = "[]",
            CreatedTimestamp = GetInt64(payload, "completedAt") ?? DateTimeOffset.UtcNow.ToUnixTimeMilliseconds(),
            ModelName = null,
            TraceId = GetString(payload, "traceId"),
            SourceFactsJson = payload.GetRawText(),
            CreatedAt = ToIsoTime(...)
        };

        store.InsertObservation(record);
        logger.LogInformation("Workflow task 已入记忆：{TaskId}", record.Id);
        return true;
    }
}
```

```csharp
public async Task TryGenerateAsync(string taskId, string finalReport)
{
    var task = _taskRepo.Get(taskId);
    if (task is null || task.IsTitleAutoGenerated == 0) return;
    if (string.IsNullOrWhiteSpace(finalReport)) return;

    try
    {
        // 通过公共组件解析（阶段 2B 已抽出 IChatCompactionClientResolver）
        var client = _resolver.Resolve();
        if (client is null) return;

        using var _ = (client as TokenTrackingChatClient)?.SuppressUsage();

        var prompt = new List<AIChatMessage>
        {
            new(ChatRole.System,
                "你是工作任务标题生成助手。根据任务描述和最终报告，提取关键词并总结成一个简短的任务标题。" +
                $"要求：不超过 {_options.TaskTitleMaxLengthForLlm} 个字，不要引号，不要标点，直接输出标题文本。"),
            new(ChatRole.User,
                $"任务描述：{task.InitialInput.Truncate(200)}\n\n" +
                $"最终报告：{finalReport.Truncate(800)}")
        };

        var completion = await client.GetResponseAsync(prompt);
        var title = completion?.Text?.Trim();
        if (string.IsNullOrWhiteSpace(title)) return;

        title = title.Trim('"', '"', '"', '「', '」', '\'').Truncate(_options.TaskTitleMaxLength);

        var now = DateTimeOffset.UtcNow.ToUnixTimeMilliseconds();
        _taskRepo.UpdateTitle(taskId, title, isAutoGenerated: false, lastActiveTimestamp: now);

        _publisher.Publish(WorkflowEvents.OnWorkflowTaskTitleUpdated,
            new WorkflowTaskTitleUpdatedArgs(...) { Title = title, Source = "llm" });

        _logger.LogInformation("Workflow 任务 {TaskId} 标题已由 AI 生成：{Title}", taskId, title);
    }
    catch (Exception ex)
    {
        _logger.LogWarning(ex, "为 Workflow 任务 {TaskId} 生成 AI 标题失败，保留占位标题。", taskId);
        // 不更新 IsTitleAutoGenerated，下次重启 / 手动触发可重试
    }
}
```

**关键设计点**：

- 复用阶段 2B 抽出的 `IChatCompactionClientResolver`，与 Chat 同源
- `(client as TokenTrackingChatClient)?.SuppressUsage()` 隔离 usage 上报
- 标题生成失败仅 LogWarning，保留占位、保持 `IsTitleAutoGenerated=1` 待重试
- 发独立事件 `OnWorkflowTaskTitleUpdated`，UI 局部刷新列表项
