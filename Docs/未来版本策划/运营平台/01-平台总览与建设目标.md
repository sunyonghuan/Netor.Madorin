# 01. 平台总览与建设目标

## 1. 项目定位

Madorin 运营平台用于公开分发、交易、审核和运营 Madorin 生态内的四类核心资产：

- **插件（Plugin）**：扩展本地能力、系统能力、第三方服务能力的可安装组件。
- **技能（Skill）**：面向特定场景的提示词、工作流、知识包、工具调用规范和任务策略。
- **智能体（Agent）**：具备角色设定、工具权限、长期任务流程、领域能力封装的可复用 AI 工作单元。
- **解决方案（Solution）**：由多个插件、技能、智能体、配置模板、部署说明组合而成的一站式能力包。

平台目标不是单纯的“插件下载站”，而是建立一个可持续运营的 **AI 能力市场 + 创作者分成平台 + 企业订阅管理平台 + 官方生态分发中心**。

## 2. 建设目标

### 2.1 用户侧目标

- 用户可以注册、登录、购买、订阅、续费和管理已获得的插件、技能、智能体和解决方案。
- 用户可以在本地 Madorin 客户端中一键安装、升级、禁用、卸载已购内容。
- 用户可以查看自己的订阅状态、订单、发票、授权设备、使用记录和到期提醒。
- 用户可以对插件、技能和智能体进行评分、评论、收藏、举报和反馈。

### 2.2 创作者侧目标

- 创作者可以上传自己的插件、技能、智能体或解决方案。
- 创作者可以设置定价模式、订阅周期、试用策略、版本说明和售后说明。
- 创作者可以查看销售数据、订阅人数、退款率、评分、审核状态和收益明细。
- 平台按规则抽成，并向创作者结算收益。

### 2.3 平台侧目标

- 平台拥有总管理中心，用于内容审核、用户管理、订阅管理、资金管理、财务管理、风控管理和运营配置。
- 平台可以支持官方内容、第三方内容、企业私有内容、定向授权内容。
- 平台可以建立标准化插件包、技能包、智能体包和解决方案包格式。
- 平台预留后续企业版、团队版、私有市场、渠道分销、增值服务能力。

## 3. 平台角色设计

### 3.1 普通用户

普通用户是内容消费者，主要权限包括：

- 浏览公开市场内容。
- 搜索插件、技能、智能体和解决方案。
- 免费安装或付费订阅。
- 管理自己的订阅、订单、设备授权和本地安装状态。
- 评价、评论、收藏、举报内容。
- 申请退款或提交售后工单。

### 3.2 创作者 / 开发者

创作者是内容提供方，主要权限包括：

- 创建开发者档案。
- 提交实名、资质、收款账户和税务信息。
- 上传插件、技能、智能体、解决方案。
- 管理版本、定价、试用、授权范围和分发范围。
- 查看收益、销售数据、订阅留存、用户反馈。
- 提交内容复审、申诉和下架申请。

### 3.3 企业用户

企业用户面向团队或组织，主要权限包括：

- 创建企业空间。
- 管理成员、角色、团队授权和席位。
- 统一购买订阅和分配插件、技能、智能体。
- 查看团队使用情况、费用、授权审计。
- 使用企业私有市场或定向采购解决方案。

### 3.4 审核员

审核员负责平台内容合规和安全审查，主要权限包括：

- 查看待审核内容。
- 检查插件包结构、权限声明、依赖项、脚本风险、网络访问、敏感能力。
- 审核技能和智能体中的提示词安全、工具权限、数据访问范围。
- 审核解决方案组合包的依赖关系和授权边界。
- 执行通过、驳回、要求修改、冻结、下架等操作。

### 3.5 平台管理员

平台管理员负责系统治理，主要权限包括：

- 用户管理、角色管理、权限管理。
- 商品管理、分类管理、标签管理、推荐位管理。
- 订阅套餐管理、价格管理、折扣管理。
- 订单管理、支付管理、退款管理、财务结算。
- 审核策略、风控策略、黑名单、白名单、申诉处理。
- 公告、运营活动、版本策略、平台参数配置。
