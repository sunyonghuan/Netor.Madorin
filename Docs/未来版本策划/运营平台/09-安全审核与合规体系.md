# 09. 安全审核与合规体系

## 1. 插件安全

- 静态扫描：敏感 API、网络访问、文件系统访问、进程启动、注册表访问。
- 动态沙箱：运行行为检测。
- 权限最小化：插件只能申请必要权限。
- 签名机制：未签名插件默认禁止市场安装。
- 黑名单机制：恶意作者、恶意包哈希、恶意域名。

## 2. AI 内容安全

- 技能和智能体需要检查提示词注入风险。
- 检查是否诱导越权调用工具。
- 检查是否包含敏感凭据、违法内容、侵权内容。
- 高风险智能体需要额外审核工具权限。

## 3. 交易安全

- 支付回调签名校验。
- 防重复回调。
- 订单幂等处理。
- 退款冻结期。
- 高频交易风控。
- 虚假交易检测。

## 4. 隐私合规

- 用户数据最小化采集。
- 明确隐私政策和开发者协议。
- 插件必须声明收集和传输的数据类型。
- 用户可以查看和撤销授权。
- 企业数据和个人数据隔离。

## 5. 自动审核

自动审核包括：

- 包结构校验。
- 版本号校验。
- manifest 校验。
- 哈希计算。
- 病毒扫描。
- 敏感 API 扫描。
- 权限声明匹配。
- 依赖漏洞扫描。
- AI 提示词风险扫描。

## 6. 人工审核

人工审核包括：

- 功能描述是否真实。
- 权限声明是否合理。
- 是否存在误导性营销。
- 是否侵犯版权或商标。
- 是否包含敏感、违法或违规内容。
- 是否符合平台 UI 和上架规范。

## 7. 上架后监管

- 用户举报。
- 评分异常检测。
- 崩溃率监控。
- 退款率监控。
- 安全事件响应。
- 版本回滚。
- 紧急下架。

## 8. 法务协议

需要准备以下协议：

- 用户服务协议。
- 隐私政策。
- 开发者协议。
- 付费订阅协议。
- 退款政策。
- 内容审核规范。
- 插件安全规范。
- 知识产权投诉规则。
- 企业服务条款。

重点约束：

- 创作者保证上传内容拥有合法权利。
- 插件不得未经授权收集用户数据。
- 平台保留下架违规内容的权利。
- 收费内容需要明确试用、退款和服务范围。
- 企业用户需要单独约定数据边界和 SLA。
