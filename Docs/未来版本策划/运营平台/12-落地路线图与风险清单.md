# 12. 落地路线图与风险清单

## 1. 分阶段落地计划

### Phase 1：基础市场与账号体系

目标：完成最小可用市场。

- 用户注册登录。
- 商品列表和详情。
- 免费内容下载。
- 基础插件 / 技能 / 智能体包格式。
- 后台内容录入。
- 客户端安装入口。

### Phase 2：订阅付费与用户中心

目标：完成商业闭环。

- 支付接入。
- 订阅套餐。
- 订单系统。
- 用户订阅中心。
- 授权校验。
- 发票和退款基础能力。

### Phase 3：创作者中心与审核系统

目标：开放第三方生态。

- 开发者入驻。
- 内容上传。
- 自动审核。
- 人工审核。
- 上架流程。
- 创作者数据看板。

### Phase 4：财务结算与风控

目标：支持平台化运营。

- 分成规则。
- 结算单。
- 提现。
- 对账。
- 风险订单。
- 违规内容治理。

### Phase 5：企业版与解决方案市场

目标：提高客单价和企业价值。

- 企业空间。
- 席位管理。
- 企业订阅。
- 私有市场。
- 解决方案组合售卖。
- 企业合同和 SLA。

## 2. MVP 范围建议

第一版建议只做必要闭环：

- 用户注册登录。
- 市场列表、详情、搜索。
- 免费和付费订阅两种商品。
- 用户订阅中心。
- 插件、技能、智能体三类资产的上传和后台录入。
- 简单人工审核。
- 客户端下载、签名校验、安装。
- 基础订单和支付回调。
- 管理后台：用户、商品、订单、审核。

暂缓：

- 创作者自动提现。
- 企业私有市场。
- 复杂分销。
- 按调用量计费。
- 高级数据分析。
- 多区域部署。

## 3. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 恶意插件 | 用户设备受损、平台信誉受损 | 签名、沙箱、权限、审核、黑名单 |
| 低质量内容泛滥 | 用户体验下降 | 入驻门槛、评分、退款率、推荐算法约束 |
| 支付和退款纠纷 | 财务损失 | 明确退款政策、冻结期、人工复核 |
| 创作者违规上传 | 法务风险 | 实名认证、协议约束、处罚机制 |
| 授权被破解 | 收入损失 | 在线校验、本地签名、设备绑定、异常检测 |
| 企业数据泄露 | 高严重度事故 | 数据隔离、权限最小化、审计日志 |
| 平台运营复杂度高 | 开发周期长 | 分阶段上线，先 MVP 后扩展 |

## 4. 需要进一步确认的问题

- 平台是先做 Web 独立站，还是先内嵌到 Madorin 客户端？
- 支付方式优先支持微信、支付宝、Stripe，还是仅国内支付？
- 插件上传是否一开始就开放第三方，还是先白名单邀请？
- 插件是否必须经过沙箱运行，还是先使用签名 + 审核控制？
- 创作者结算是否需要自动提现，还是早期由人工财务处理？
- 企业空间是否进入第一版？
- 是否需要支持私有化部署市场？
