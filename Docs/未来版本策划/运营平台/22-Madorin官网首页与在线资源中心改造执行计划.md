# 22. Madorin 官网首页与在线资源中心改造执行计划

## 目标

把 `Netor.Cortana.Platform.Web` 从“市场 MVP 页面”升级为正式的 Madorin 官方网站与资源中心入口。

第一阶段先完成：

- 官网首页营销化改版，突出下载转化。
- 全站前台视觉重构，符合软件行业官网气质。
- 大屏与手机屏幕响应式适配。
- 资源中心保留现有市场、详情、订阅和下载入口。
- 软件截图先使用界面占位符，后续替换为真实截图。

## 首页表达重点

首页只讲用户马上能感知的价值：

- Madorin 不是普通 AI 聊天工具，而是个人 AI 公司操作系统。
- 用户像 CEO 一样提出目标、审批方向、选择最终方案。
- AI 智能体像员工一样承担岗位，组成技术、市场增长、客服、产品策划、运营管理、能力扩展等部门。
- 会议模式让多个专家智能体参与讨论、反驳、表决，并总结成会议纪要、执行清单和最终方案。
- 工作流模式把方案继续推进成文档、脚本、待办、检查清单和可交付结果。
- 新手不需要懂模型、智能体、提示词或运行原理，打开软件就能从普通目标开始使用。
- 技能、插件和脚本让软件持续扩展，逐步成长为用户自己的专属公司系统。

文案原则：

- 短句。
- 不写空泛愿景。
- 站在创业者、小团队、独立开发者和自由职业者的视角说话。
- 把技术能力翻译成业务场景，不堆 AI 术语。
- 直接命中“缺人手、不会配置、不懂 AI、方案落不了地、事情没人推进”等痛点。
- 首页只负责介绍软件价值和下载转化，不展示资源中心内容。

## 视觉方向

- 品牌展示名统一使用 `Madorin`。
- 首屏使用深色软件官网风格，突出产品名、下载按钮和界面占位图。
- 页面背景使用克制的深浅分区，不使用粗糙渐变卡片堆叠。
- 卡片、按钮、导航、筛选和详情页统一为现代 SaaS / 开发工具官网风格。
- 添加必要动效：首屏浮动、卡片 hover、能力流动线、渐入动画。
- 动效只服务注意力，不做干扰阅读的花活。
- 首页版面可以适当拉长，首屏必须让用户立刻理解“一个人也能指挥一家公司”的核心卖点。

## 在线资源中心判断

在线技能实时加载可以实现，也有必要做，但不应做成每次都完全在线读取。

推荐模式：

```text
线上资源中心 = 权威源
本地 = 缓存
客户端 = manifest 校验 + 订阅鉴权 + 按需下载/刷新
```

加载流程：

```text
客户端登录
  -> 保存 token
  -> 请求 skill manifest
  -> 服务端校验订阅
  -> 返回版本、hash、签名、下载地址或内容
  -> 客户端校验 hash/signature
  -> 写入本地缓存
  -> 加载技能
```

当前已有 `Asset`、`AssetVersion`、`PricingPlan`、`Subscription`、`DownloadRecord`，可以承载第一版资源授权。后续需要补 API、客户端账号设置、token、签名和缓存策略。

## 后续 API 建议


## 执行步骤

1. 重构官网首页、下载页和共享布局。
2. 统一市场首页、资源列表、资源卡片和详情页视觉。

4. 构建验证 Web 项目。
5. 使用浏览器检查桌面和手机宽度下的首屏、资源卡片和下载页。

- 首页语言精炼，没有“后续占位”“MVP 当前能力”等上线前措辞。
- 大屏首屏有产品界面占位，手机屏幕不重叠、不横向滚动。
- 市场和资源页视觉与官网一致。
- 现有登录、市场、资源详情、下载入口保留。
