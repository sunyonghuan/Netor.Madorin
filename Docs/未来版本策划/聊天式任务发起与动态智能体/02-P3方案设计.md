# 02 — P3 方案设计：流程化可视化 + 文件夹附件（体验完整版）

> **阶段定位**：P3 是体验完整版，结束后用户能实时看到任务执行的全过程（思考 / 工具调用 / 子智能体协作），并能用文件夹作为附件。
>
> **工期**：2 周（2-3 个 commit）
>
> **前置依赖**：P2 完成（聊天式输入 + 动态 Agent + Plan 审批 + 保存常用）
>
> **不包含**：单元测试覆盖（→ P4 稳定性 / 可观测性阶段）

---

## 0. 范围 / 边界

### 0.1 IN（P3 必做）

| 模块 | 内容 |
|---|---|
| 工作流可视化 | 时间线视图（左侧时间轴 + 智能体名 + 内容 + 思考 / 工具调用折叠）|
| 群聊可视化 | 聊天气泡（与 Chat 一致 + 头像 + 时间 + 思考气泡 + 工具气泡）|

| 滚动性能 | `VirtualizingStackPanel` 让 100+ steps 列表不卡 |
| 富文本渲染 | Agent 输出 markdown / 代码块 / 表格（与 Chat 输出一致）|
| 文件夹附件（核心） | Workflow / GroupChat / Chat 三处统一支持文件夹拖放 + 文件夹选择器 |
| 右键发送 | WorkspaceExplorer 右键菜单加「发送到对话」/「发送到工作流」/「发送到群聊」|

### 0.2 OUT（不做）

- ❌ 嵌套工具调用的链式折叠（如 `read_file → ai_summarize → write_file`，→ 远期）
- ❌ 文件夹大小硬上限（决策 TP-9 B2：不限大小但自动过滤）
- ❌ 富文本编辑（用户输入仍是纯文本 + Markdown）
- ❌ Agent 头像自定义图片（用 Emoji 顶替，与 EmptyState C1 同款降级方案）

## 1. 架构选型

### 1.1 工作流可视化：时间线视图

| 选项 | 描述 | 工作量 | 风险 |
|---|---|---|---|
| **A（选用）** | 自定义 ItemsControl + DataTemplate 渲染步骤 | 1 周 | 低 |
| B | 引入第三方 Timeline 控件（如 Avalonia.Toolkit）| 0.5 周 | 中（引入依赖 + AOT 兼容性风险）|
| C | 直接复用 Chat 消息流（气泡 + Avatar）| 0.5 周 | 中（语义不符：工作流是时间线，不是对话）|

**选 A 的理由**：
- 工作流的语义就是时间线（时间 + 智能体 + 内容），不是对话
- 不引入第三方依赖，AOT 兼容性零风险
- DataTemplate 可在 axaml 编译期静态分析（Avalonia 11 严格模式）

### 1.2 群聊可视化：聊天气泡

| 选项 | 描述 | 工作量 | 风险 |
|---|---|---|---|
| **A（选用）** | 直接复用 Chat ChatMessageView 90% | 5 天 | 低 |
| B | 自定义 GroupChatMessageView（与 Chat 解耦）| 1 周 | 中（复制 80% 代码）|

**选 A 的理由**：
- 群聊就是多个 agent 平等对话，与 Chat 模式语义一致
- 唯一差异是 Agent 名前缀（"🤖 code_reviewer 14:23"），CSS 类可控

### 1.3 文件夹附件：递归枚举 + 过滤

| 选项 | 描述 | 工作量 | 风险 |
|---|---|---|---|
| **A（选用）** | 选择文件夹 → `Directory.EnumerateFiles(recursive=true)` + 过滤规则 | 5 天 | 中 |
| B | 选择文件夹 → 用户手动勾选要包含的文件 | 2 周 | 低（但用户体验更复杂）|
| C | 选择文件夹 → 上传整个 zip | 1 周 | 中（解压会卡）|

**选 A 的理由**：
- 满足"一个文件夹整体发送"的核心痛点
- 过滤规则可配置，覆盖 90% 场景（用户痛点：`node_modules` 那种）
- B 多了一步操作，违背"省事"的初心

#### A. AttachmentInfo 扩展

```csharp
public sealed record AttachmentInfo(
    string Path,
    string Name,
    string MimeType,
    bool IsFolder = false,           // ★ P3 新增
    int FileCount = 0,               // ★ P3 新增（文件夹下文件数）

```

- 文件：`IsFolder=false / FileCount=0 / TotalBytes=文件大小`
- 文件夹：`IsFolder=true / FileCount=扫描结果 / TotalBytes=总和`

#### B. FolderAttachmentScanner

```csharp
public sealed class FolderAttachmentScanner
{
    public FolderAttachmentResult Scan(string folderPath, FolderAttachmentScannerOptions options);
}

public sealed record FolderAttachmentResult(
    string RootPath,
    IReadOnlyList<string> IncludedFiles,    // 通过过滤的文件
    IReadOnlyList<string> ExcludedReasons,  // 被过滤的原因（"node_modules", "二进制" 等）
    long TotalBytes,
    int FileCount);

public sealed class FolderAttachmentScannerOptions
{
    /// <summary>排除规则（gitignore 风格 glob）。</summary>
    public IReadOnlyList<string> ExcludePatterns { get; init; } = [
        "node_modules/**",
        ".git/**",
        "bin/**",
        "obj/**",
        "**/*.dll",
        "**/*.exe",
        "**/*.so",
        "**/*.dylib",
        "**/*.zip",
        "**/*.tar.gz",
        "**/*.pdb",
    ];

    /// <summary>最大递归层数（防止符号链接循环）。</summary>
    public int MaxDepth { get; init; } = 10;

    /// <summary>单文件最大大小（超过自动过滤，避免内存炸）。</summary>
    public long MaxFileBytes { get; init; } = 10 * 1024 * 1024;  // 10 MB
}
```

```csharp
private void OnDrop(object? sender, DragEventArgs e)
{
    var files = e.DataTransfer.TryGetFiles();
    foreach (var item in files)
    {
        var path = item.Path?.LocalPath;
        if (Directory.Exists(path))
        {
            // ★ 新增：文件夹处理
            var scanner = App.Services.GetRequiredService<FolderAttachmentScanner>();
            var result = scanner.Scan(path, new FolderAttachmentScannerOptions());
            _attachments.Add(new AttachmentInfo(
                Path: result.RootPath,
                Name: Path.GetFileName(result.RootPath),
                MimeType: "application/x-folder",
                IsFolder: true,
                FileCount: result.FileCount,
                TotalBytes: result.TotalBytes));
        }
        else if (File.Exists(path))
        {
            // 现有文件处理逻辑
        }
    }
}
```

#### E. AIAgent 接收文件夹附件

```csharp
async Task<string> InvokeAgentWithAttachmentsAsync(
    string query,
    string[]? attachmentPaths,
    string[]? attachmentDescriptions,
    CancellationToken ct)
{
    var contents = new List<AIContent> { new TextContent(query) };

    foreach (var path in attachmentPaths ?? [])
    {
        if (Directory.Exists(path))
        {
            // ★ P3 新增：文件夹 → 递归扫描 → 多个 TextContent
            var scanner = App.Services.GetRequiredService<FolderAttachmentScanner>();
            var result = scanner.Scan(path, new FolderAttachmentScannerOptions());

            contents.Add(new TextContent($"\n[文件夹: {Path.GetFileName(path)} ({result.FileCount} 文件)]"));
            foreach (var file in result.IncludedFiles)
            {
                var relativePath = Path.GetRelativePath(path, file);
                contents.Add(new TextContent($"\n[{relativePath}]({file})"));
            }
        }
        else if (File.Exists(path))
        {
            // 现有文件处理
        }
    }

    return await subAgent.InvokeAsync(contents, ct);
}
```

## 4. 风险与规避

| # | 风险 | 等级 | 缓解 |
|---|---|---|---|
| 1 | 100+ steps 滚动卡顿 | 中 | VirtualizingStackPanel + 决策 1.1 A 自定义 ItemsControl 控制 DataTemplate 复杂度 |
| 2 | 富文本渲染性能（大量 markdown）| 中 | 复用 Chat 模式的 MarkdownRenderer.cs（已经过实战验证）|
| 3 | 文件夹扫描卡 UI（大文件夹）| 中 | `FolderAttachmentScanner.ScanAsync` 在后台线程运行 + Snackbar 进度提示 |
| 4 | 用户拖入巨大文件夹（如 C:\） | 高 | MaxDepth + MaxFileBytes 双重限制 + 扫描结果 > 10000 文件时 Toast 警告 |
| 5 | 符号链接循环 | 中 | MaxDepth + `Directory.EnumerateFiles` 不跟随符号链接（默认行为）|
| 6 | 时间线自动滚动与用户手动滚动冲突 | 低 | 检测用户滚动距底部超过 50px 时停止自动滚动 |
| 7 | 群聊气泡与 Chat 气泡视觉冲突 | 低 | 共用 `message-bubble` 基类 + 群聊额外 `groupchat-msg` 修饰类 |

**回退方案**：
- P3-1 独立 commit，若可视化有 bug 可单 revert，恢复 EmptyState 占位（后端事件流仍工作）
- P3-2 独立 commit，若文件夹有 bug 可单 revert，恢复"只接收文件"行为
