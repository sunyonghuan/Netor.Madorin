# 21. 个人中心与订阅周期补充设计

## 1. 问题背景

当前个人前台已经具备登录、订阅、下载和订单页面，但用户中心不能只停留在“我的订阅”。个人用户最基础的能力还应包含：

- 个人资料维护。
- 联系方式维护。
- 订阅周期展示。
- 即将到期提醒。
- 过期订阅处理。
- 插件、解决方案等资源按月付费订阅。

## 2. 当前数据库是否缺表

结论：**第一阶段不一定缺表，但缺字段语义和业务规则说明。**

当前已有核心表：

- `Account`：用户账号，继承 `AccountBase`，已有用户名、邮箱、手机、昵称、真实姓名等基础字段。
- `PricingPlan`：价格方案，已有 `PlanType`、`Price`、`Currency`、`DurationDays`。
- `Subscription`：订阅记录，已有 `StartedAtUtc`、`ExpiresAtUtc`、`CanceledAtUtc`、`Status`。
- `Order`：订阅订单。
- `DownloadRecord`：下载记录。

因此：

- 个人资料：可以直接先用 `Account` 字段。
- 月付订阅：可以用 `PricingPlan.PlanType = Monthly` + `DurationDays = 30`。
- 年付订阅：可以用 `PricingPlan.PlanType = Yearly` + `DurationDays = 365`。
- 到期判断：可以用 `Subscription.ExpiresAtUtc`。

## 3. 当前设计缺口

### 3.1 个人中心缺口

必须补齐：

- 个人资料页面。
- 资料保存接口。
- 个人中心首页展示即将到期订阅。
- 我的订阅显示资源类型、计费周期、剩余天数。

后续再补：

- 修改密码。
- 邮箱验证。
- 手机验证。
- 头像上传。
- 登录日志。
- 账号注销。

### 3.2 订阅周期缺口

当前 `PricingPlan.DurationDays` 可以表达周期，但文档需要明确：

| PlanType | DurationDays | 说明 |
|---|---:|---|
| Free | 0 | 长期有效，代码可转为远期时间 |
| Monthly | 30 | 月付订阅，第一阶段按 30 天处理 |
| Yearly | 365 | 年付订阅，第一阶段按 365 天处理 |
| OneTime | 0 或约定值 | 买断或一次性授权，后续再细化 |

### 3.3 插件和解决方案月付规则

插件、解决方案、技能、智能体都统一使用 `Asset` 表，因此付费规则不需要拆表。

建议：

- 插件支持 `Monthly` / `Yearly` / `OneTime`。
- 解决方案支持 `Monthly` / `Yearly`。
- 技能和智能体也可以使用相同规则。
- 同一个资源可存在多个有效价格方案。

## 4. 订阅状态计算规则

订阅实际状态应由两部分共同决定：

```text
显示状态 = Subscription.Status + ExpiresAtUtc 与当前时间比较
```

建议规则：

| 数据状态 | 时间状态 | 展示 |
|---|---|---|
| Active | ExpiresAtUtc > now | 有效 |
| Active | ExpiresAtUtc <= now | 已过期 |
| Canceled | 任意 | 已取消 |
| Expired | 任意 | 已过期 |

第一阶段可以在查询时计算展示状态；后续可增加定时任务批量把已到期订阅标记为 `Expired`。

## 5. 后续可能需要新增的字段

第一阶段可以不加，正式订阅体系建议补：

### 5.1 `PricingPlan`

- `BillingCycle`：计费周期，Monthly / Yearly / OneTime。
- `AutoRenewEnabled`：是否支持自动续费。
- `TrialDays`：试用天数。

当前可先复用 `PlanType` 和 `DurationDays`。

### 5.2 `Subscription`

- `RenewedFromSubscriptionId`：续订来源。
- `NextBillingAtUtc`：下次扣费时间。
- `AutoRenewEnabled`：用户是否开启自动续费。
- `LastRenewedAtUtc`：最后续费时间。

第一阶段不接真实支付和自动扣费，暂不强制新增。

本次已补：

## 7. 下一步建议

下一步优先级：

1. 补“取消订阅”操作。
2. 补“续订”入口，过期或即将到期订阅可以重新创建订单。
3. 增加后台价格方案维护时的月付/年付引导。
4. 后续再考虑 `AutoRenewEnabled`、`NextBillingAtUtc` 等正式自动续费字段。
