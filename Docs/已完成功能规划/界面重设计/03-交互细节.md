## 1. 模式切换状态机

### 1.1 状态枚举

```csharp
public enum WorkMode
{
    Chat,        // 对话
    Workflow,    // 工作流（SubMode in magentic/parallelanalysis）
    GroupChat    // 群聊（SubMode = groupchat）
}
```

### 1.2 状态切换矩阵

| 起始 | 触发 | 目标 | 守卫条件 | 副作用 |
|---|---|---|---|---|
| Chat | 点工作流 tab | Workflow | InputBox 无未提交内容 | 直接切 |
| Chat | 点工作流 tab | Chat（保留） | InputBox 有未提交内容 + 用户点"取消" | 不切 |
| Chat | 点工作流 tab | Chat（保留） | InputBox 有未提交内容 + 用户点"保留内容" | 切走，输入框文本暂存到 ChatDraftService |
| Chat | 点工作流 tab | Workflow（丢弃） | InputBox 有未提交内容 + 用户点"丢弃并切换" | InputBox.Text = "" |
| Workflow → Chat | 点对话 tab | Chat | 有 ChatDraft | InputBox.Text 恢复为草稿 |
| Workflow → Chat | 点对话 tab | Chat | 无 ChatDraft | InputBox.Text = "" |
| Workflow → GroupChat | 点群聊 tab | GroupChat | 任意（任务在后台跑不影响） | 直接切 |
| 任意 → 任意 | 点顶栏「最近 ▼」选项 | 目标模式 + 选中目标 item | 同上守卫规则 | 自动激活目标模式 + 切到目标 |

### 1.3 守卫条件「有未提交内容」定义

```csharp
bool HasUnsavedChatInput =>
    !string.IsNullOrWhiteSpace(InputBox.Text)
    || AttachmentList.Items.Count > 0;
```

附件存在 → 也算未保存（用户已选择文件准备发送，切走应该提醒）。

### 1.4 ChatDraftService 设计

```csharp
public sealed class ChatDraftService
{
    // 内存级单例，进程退出时丢弃；不入数据库
    private string? _draftText;
    private List<AttachmentInfo> _draftAttachments = [];

    public void Save(string text, IEnumerable<AttachmentInfo> attachments) { /*...*/ }
    public (string?, List<AttachmentInfo>) Restore() { /*...*/ }
    public bool HasDraft => _draftText is not null;
    public void Clear() { /*...*/ }
}
```

**生命周期**：
- 切走时 `Save`
- 切回时 `Restore`，并 `Clear`（一次性恢复）
- 进程退出 → 内存释放，不持久化（设计取舍：避免敏感对话留盘）

## 2. 模式切换确认对话框

### 2.1 触发条件

仅当下面**所有**条件都满足时弹出：

1. 当前 CurrentMode == Chat
2. HasUnsavedChatInput == true
3. 用户触发切换到 Workflow 或 GroupChat

工作流 / 群聊 → 对话切换**永不弹**（无未保存概念）。

### 2.2 对话框规格

```
┌─ 切换工作模式 ──────────────────────────────────────┐
│                                                      │
│  当前对话有未发送的内容，是否丢弃？                   │
│                                                      │
│  ┌────────────────────────────────────────────────┐  │
│  │ "<前 50 字预览，超出截断>..."                  │  │
│  └────────────────────────────────────────────────┘  │
│                                                      │
│  3 个附件                                            │
│                                                      │
│              [取消]  [保留内容]  [丢弃并切换]        │
│                                                      │
└──────────────────────────────────────────────────────┘
```

| 元素 | 规格 |
|---|---|
| 对话框尺寸 | 440 × 自适应 |
| 标题 FontSize | 16 SemiBold |
| 提示 FontSize | 13 TextBrush |
| 预览块 Background | Surface0Brush |
| 预览块 Padding | 12,8 |
| 预览块 CornerRadius | 4 |
| 预览块 MaxHeight | 80（超出加滚动条） |
| 附件数显示 | 仅当 > 0 显示 |
| 按钮区 HorizontalAlignment | Right |
| 按钮间距 | 8 |

### 2.3 按钮行为

| 按钮 | Class | 行为 |
|---|---|---|
| 取消 | btn-secondary | DialogResult=Cancel；不切换 |
| 保留内容 | btn-secondary | 调用 `ChatDraftService.Save()`；切换到目标模式 |
| 丢弃并切换 | btn-danger | 清空 InputBox + AttachmentList；切换到目标模式 |

按 ESC = 取消，按 Enter = 取消（保守默认，避免误丢失数据）。

### 2.4 实现要点

- 使用 Avalonia `Window.ShowDialog<T>(parent)` 模态弹窗
- 返回 `DialogResult` 枚举 (Cancel / Save / Discard)
- 调用方根据结果决定后续动作
- 对话框样式复用 `btn-secondary` / `btn-danger` 已有定义

## 3. 顶栏「最近 ▼」交互

### 3.1 打开 / 关闭

- 点击按钮：打开下拉
- 点击下拉外区域 / ESC：关闭下拉（IsLightDismissEnabled=true）
- 点击下拉项：执行切换 + 关闭下拉

### 3.2 下拉项内容（每模式 10 条）

| 区域 | 内容 |
|---|---|
| Tab 头部 | "最近会话"/"最近任务"/"最近群聊" 标签（FontSize=11 Subtext） |
| 列表 | 最近 10 条，按 LastActiveTimestamp DESC |
| 列表项 | 标题（FontSize=12，截断 240 字符）+ 相对时间（FontSize=11 Subtext 右对齐） |
| 底部 | "查看全部" 链接 → 关闭下拉 + 自动激活左侧 Tab2 |

### 3.3 相对时间格式

| 范围 | 格式 |
|---|---|
| < 1 分钟 | 刚刚 |
| 1-59 分钟 | N 分钟前 |
| 1-23 小时 | N 小时前 |
| 1-6 天 | N 天前 |
| ≥ 7 天 | YYYY-MM-DD |

### 3.4 模式切换 + 选中联动

点击下拉项时：

1. 如果目标模式 ≠ 当前模式：先走"模式切换确认对话框"流程
2. 切换成功后激活目标模式
3. 选中目标 item（设置 SelectedItem）
4. 左侧 Tab2 自动滚动到该 item

时序保证：先切模式，再选 item（防止 SelectedItem 找不到上下文）。

## 4. 左侧 Tab 联动时序

### 4.1 顶栏 tab 切换触发的级联

```
用户点击「工作流」tab
   ↓
MainWindowVm.CurrentMode = WorkMode.Workflow
   ↓ PropertyChanged
   ├─→ LeftPanel.Tab2Header (绑定到 ModeBoundLabel)
   │   ├─ Chat → "会话历史"
   │   ├─ Workflow → "工作流任务"
   │   └─ GroupChat → "群聊任务"
   │
   ├─→ LeftPanel.Tab2Content (Switch by mode)
   │   ├─ Chat → ChatHistoryPanel
   │   ├─ Workflow → TaskListPanel (filter=workflow)
   │   └─ GroupChat → TaskListPanel (filter=groupchat)
   │
   ├─→ LeftPanel.SearchPlaceholder (绑定)
   ├─→ LeftPanel.NewItemButtonText (绑定)
   ├─→ MainContent.ActiveView (Switch)
   └─→ TitleBar.RecentDropdownLabel (绑定)
```

### 4.2 切换激活 tab 的策略

| 用户当前激活 | 切换顶栏模式后 | 行为 |
|---|---|---|
| Tab1 (文件目录) | 工作流 | 保持 Tab1 激活（用户在看文件目录，不打断） |
| Tab2 (会话历史) | 工作流 | 自动切到 Tab2（"工作流任务" 列表，承接用户的"看列表"意图） |
| Tab1 | 群聊 | 保持 Tab1 |
| Tab2 | 群聊 | 切到 Tab2（群聊任务列表） |

**核心规则**：用户在看文件目录时不动文件目录；用户在看列表时跟着模式切列表。

### 4.3 状态保留

| 状态 | 是否保留 |
|---|---|
| 左侧 Tab1 文件目录展开节点 | 保留 |
| 左侧 Tab2 列表搜索关键词 | **不保留**（每个模式独立的搜索） |
| 左侧 Tab2 列表滚动位置 | 不保留（切回时回顶） |
| 左侧 Tab2 选中项 | **保留**（用户切走再切回，原选中还在） |
| Chat 输入框文本 | 走 ChatDraftService 流程 |
| Chat 附件 | 走 ChatDraftService 流程 |

## 6. 焦点管理

### 6.1 启动焦点

应用启动后默认焦点：
- 对话模式：InputBox
- 工作流 / 群聊模式：左侧 Tab2 列表（或新建按钮如果列表空）

### 6.2 Tab 切换后焦点

| 切换情境 | 焦点目标 |
|---|---|
| Chat → Workflow（首次） | 左侧 "+ 新建工作流任务" 按钮 |
| Chat → Workflow（已有任务且选中） | 主区详情面板的取消按钮（如果运行中）/ 任务标题（如果完成） |
| Chat → GroupChat | 同 Workflow |
| Any → Chat | InputBox |
| 列表项点击 | 主区详情面板（按 Tab 可进入操作按钮） |

### 6.3 Tab 键导航顺序

```
顶栏：[☰][对话 tab][工作流 tab][群聊 tab][最近 ▼][⚙]
   ↓
左侧顶部：[搜索框][+ 新建按钮]
   ↓
左侧列表：第一个列表项 → ↑↓ 翻选
   ↓
左侧底部：[文件目录 tab][动态 tab]
   ↓
主区：详情面板顶部按钮 → 内容区滚动
   ↓
对话模式：[输入框][发送按钮]
```

## 7. 拖拽交互

### 7.1 文件拖入

| 拖入区域 | 行为 |
|---|---|
| 输入框 | 作为附件加入（已存在） |
| 主区任意位置（对话模式） | 同输入框 |
| 主区任意位置（工作流模式 + 详情面板有任务） | 提示"工作流模式不支持文件附加，请切到对话模式" |
| 左侧文件目录 tab | 复制到当前工作目录 |
| 左侧任务列表 tab | 无效（防误操作） |

### 7.2 列表项拖拽

| 拖拽源 | 拖拽目标 | 行为 |
|---|---|---|
| 左侧文件项 | 输入框 | 引用为 # 文件 |
| 左侧会话项 | 任何位置 | 无（仅点击触发切换） |
| 左侧任务项 | 任何位置 | 无（防止意外操作任务） |

阶段 7+ 可考虑：拖拽任务到对话输入框 → 引用任务 ID。

## 8. 错误反馈

### 8.1 Toast 全局服务

替代各页面独立的 ToastBorder。

```csharp
public interface IToastService
{
    void Show(string message, ToastKind kind = ToastKind.Info, int durationMs = 3000);
}

public enum ToastKind { Info, Success, Warning, Error }
```

### 8.2 Toast 视觉规格

| Kind | Background | BorderBrush | Icon |
|---|---|---|---|
| Info | Surface1Brush | BlueBrush | ℹ️ |
| Success | Surface1Brush | SuccessBrush #0e7c3c | ✓ |
| Warning | Surface1Brush | WarningBrush #e0c074 | ⚠ |
| Error | Surface1Brush | RedBrush | ✕ |

| 属性 | 值 |
|---|---|
| 位置 | 主区顶部居中，距顶 52px |
| 宽度 | 自适应，最大 520px |
| Padding | 16,10 |
| CornerRadius | 8 |
| BoxShadow | ShadowLg `0 6 18 0 #66000000` |
| 动画 | 滑入（顶 → 当前位置 200ms）+ 滑出 |
| 默认时长 | 3000ms |
| Error 时长 | 5000ms |

### 8.3 错误对话框 vs Toast 选择

| 场景 | 选择 |
|---|---|
| 操作完成（保存/删除成功） | Toast Info / Success |
| 非阻塞错误（网络抖动/Token 超限） | Toast Warning |
| 阻塞错误（数据库损坏/AOT 兼容失败） | 错误对话框 + 提供操作 |
| 需要用户决策（覆盖文件/丢弃数据） | 确认对话框 |

## 9. 动画与过渡

### 9.1 通用过渡时长

| 类型 | 时长 | Easing |
|---|---|---|
| 微交互（hover/pressed） | 100ms | CubicEaseOut |
| 状态切换（tab/视图） | 150ms | CubicEaseInOut |
| 弹层（popup/dialog） | 200ms | CubicEaseOut |
| 滑入滑出（Toast/Banner） | 250ms | CubicEaseInOut |

### 9.2 应避免的动画

- 不要给「列表项滚动」加动画（性能损耗大）
- 不要给「文本变化」加 fade（用户期待即时反馈）
- 不要给「关键操作按钮」加 hover 缩放（会让用户觉得控件"晃"）

### 9.3 Tab 切换过渡

| 选项 | 行为 |
|---|---|
| 无过渡（推荐） | 立即切换，最快响应 |
| Fade 150ms | 视觉过渡更柔和但有延迟感 |

**默认采用无过渡**，与现有 `Panel.IsVisible` 切换一致。

## 10. 边缘场景

### 10.1 任务运行时切走

工作流/群聊任务正在运行时切走到对话模式：
- 任务**继续在后台跑**（不取消、不暂停）
- 任务完成时触发原有事件（toast 提示"任务 #N 已完成"）
- 用户随时切回工作流模式查看进度

### 10.2 多窗口并发

阶段 7+ 可考虑多 MainWindow 实例：
- 每个窗口独立的 CurrentMode 状态
- 共享同一个 IPublisher / IExecutor 单例
- ChatDraftService 改为 per-window 实例

本期**不实现多窗口**。

### 10.3 设置变更后的 UI 刷新

| 设置变更 | UI 刷新 |
|---|---|
| 主题切换（未来浅色） | 全部重绘 |
| 字号设置 | 全部 FontSize 重计算 |
| 工作目录切换 | 文件目录 tab 重载 |
| Agent 配置变更 | 仅 NewTaskDialog 下次打开时重载 Agent 列表 |

## 11. 性能预算

| 操作 | 目标 |
|---|---|
| 模式切换响应 | < 50ms |
| 列表分页加载 30 项 | < 200ms |
| 主区视图切换 | < 100ms |

| 对话框打开延迟 | < 150ms |
