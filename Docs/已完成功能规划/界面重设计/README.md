# 界面重设计方案（总览 / 入口）

> **文档定位**：本文档是 Madorin UI 整体重设计方案的**入口与导航页**。详细技术内容已拆分到 4 篇子文档，按顺序阅读即可掌握全部方案。
> **当前版本**：v0.1 草案（2026-05-16）
> **背景**：阶段 6 完成后（commit `513a2c2`），现有 UI 累积了多个不合理点，本方案系统性重构。

---

## 0. 阅读指南

本设计源于阶段 6 收尾后的 UI 复盘。当前发现的核心问题：

1. **工作台被硬塞进中间 tab**，导致主区语义混乱（既是聊天又是任务管理）
2. **顶栏过密**：36px 内挤了 tab 切换 + 历史下拉 + 4 个图标按钮，560px 窗口撑不开
3. **右侧会话历史在 Workflow 模式下完全没意义**（任务不是会话）
4. **设计 token 落地不到位**：硬编码色值、控件样式族重复抄
5. **空状态缺失**：用户首次进入工作流/群聊模式看到一片空白，没有引导

本方案采用**布局 + token + 控件族** 三位一体的重构思路：
- 布局：左 + 主双区，左侧底 tab 切换工作目录 / 任务列表（VS 经典风格）
- token：抽 `DesignTokens.axaml` 收口色彩 / 间距 / 圆角 / 字号
- 控件族：抽 `EmptyState` / `Card` / 按钮族用户控件，避免重复定义

## 2. 决策表（带理由）

### UI-1：左侧 tab 联动模式 = L2
左侧第二个 tab 内容**随顶部模式自动切换**：
| 顶部模式 | 左侧第二 tab | 内容 |
|---|---|---|
| 对话 | 会话历史 | `ChatSessions` 列表 |
| 工作流 | 工作流任务 | `OrchestrationTask`，过滤 SubMode in [magentic, parallel] |
| 群聊 | 群聊任务 | `OrchestrationTask`，过滤 SubMode = groupchat |

**理由**：每个模式下左右侧的语义都和模式 100% 对齐，不会出现"对话模式下左侧显示工作流任务"的认知割裂。

### UI-2：右侧 = R2 干掉
原右侧会话历史挪到左侧第二 tab，整个右侧 column 取消。

**理由**：560px 窄屏 + 双侧抽屉会挤死中间主区；R2 让左右侧的语义统一为"列表"，主区永远是"详情/操作"。

### UI-3：顶栏模式 tab = B1 三个独立 tab
顶栏：`对话 | 工作流 | 群聊`

**理由**：用户从顶栏按钮就能秒切到目标模式，不用先点"工作模式"再点子模式。三个 tab 直观、单击即达。

### UI-4：顶栏「最近 ▼」= 要
快捷下拉，文案随模式变：
| 模式 | 按钮 | 下拉内容 |
|---|---|---|
| 对话 | 最近会话 ▼ | 最近 10 个会话 |
| 工作流 | 最近任务 ▼ | 最近 10 个工作流任务 |
| 群聊 | 最近群聊 ▼ | 最近 10 个群聊任务 |

**理由**：不用切左侧 tab 也能秒切到目标，覆盖"我刚刚在弄那个东西，再点一下回去"的高频场景。

### UI-5：空状态 = 居中 + 超链接
中间整个主区作为空状态画布：大图标 (64px) + 标题 (18px SemiBold) + 说明 (13px Subtext) + 超链接 (Blue `#007acc`)。

**理由**：超链接形式比按钮更轻量、无视觉重负，符合现代 SaaS（Linear / Notion / Cursor）空状态风格；与"小娜宝贝"软萌定位一致。

### UI-6：主控顶部工具栏 = 不要
主区不再有固定的"模式工具栏"，所有信息（任务标题、状态、取消按钮）都在内容区里。

**理由**：和 UI-5 配合，让主区在空状态下可以居中铺满；有内容时由内容区自身（如详情面板）决定头部样式。

### UI-7：模式切换未保存 = D2 弹确认对话框
触发条件：用户在「对话」模式下输入框有未提交文本 + 试图切换 tab。

```
┌─ 切换工作模式 ──────────────────────────────┐
│  当前对话有未发送的内容，是否丢弃？           │
│  「<前 50 字预览...>」                       │
│           [取消]  [保留内容]  [丢弃并切换]   │
└──────────────────────────────────────────────┘
```

**理由**：D1 直接丢弃太莽，D3 自动保存草稿工作量大；D2 让用户明确决定，零数据丢失风险。

工作流/群聊有正在跑的任务时**不需要**这个对话框（任务在后台跑，切 tab 不影响它）。

### UI-8：有任务时新建入口 = 候选 1（左侧顶部常驻）
左侧任务列表顶部固定一个 `+ 新建工作流任务` 入口，搜索框上方。

**理由**：UI-6 不要主控工具栏 → 排除右上角链接；候选 3「只在空状态显示」用户路径不闭环（看任务时无法直接新建）→ 候选 1 是必然推论。

### UI-9：左侧 tab 标签位置 = 底部
VS 经典风格：「文件目录 | 工作流任务」标签贴在左侧面板底边。

**理由**：顶部空间给搜索/新建/列表（视觉焦点）；tab 切换是低频操作，放底边鼠标移动短，不抢视线。

### UI-10：窗口尺寸 = 保持 560 × 980
**理由**：单手桌面右侧驻留场景下，窄竖屏比宽屏更适合。改宽屏会导致"列表-主区"比例失衡。
