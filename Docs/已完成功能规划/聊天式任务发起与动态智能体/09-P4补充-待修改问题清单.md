## 问题一：计划制定完成后，用户看不到计划内容

> **状态：✅ 已修复**

### 1.1 现象

阶段 2（计划制定）完成后，UI 只在 Feed 中追加一条事件消息，用户不知道计划内容是什么。

### 1.2 根本原因

`OrchestratorAgent.RunPlanningPhaseAsync` 解析出 `ExecutionPlan` 后直接返回，
没有通过 `onAiMessage` 把计划内容发给 UI。

### 1.3 实际修复

修改文件：`Src/Netor.Cortana.AI/TaskEngine/Agents/OrchestratorAgent.cs`

在 `RunPlanningPhaseAsync` 末尾、`return plan;` 之前，调用 `onAiMessage` 发送
Markdown 格式的计划摘要。代码位于 `OrchestratorAgent.cs` L244-262：

```csharp
if (onAiMessage is not null)
{
    var sb = new StringBuilder();
    sb.AppendLine($"好的，我已为您制定了执行计划（共 {plan.Steps.Count} 步）：");
    sb.AppendLine();
    foreach (var step in plan.Steps)
    {
        sb.AppendLine($"**步骤 {step.Sequence}：{step.Title}**");
        if (!string.IsNullOrWhiteSpace(step.Description))
            sb.AppendLine(step.Description);
        sb.AppendLine();
    }
    if (!string.IsNullOrWhiteSpace(plan.FinalGoal))
        sb.AppendLine($"最终目标：{plan.FinalGoal}");
    sb.AppendLine();
    sb.AppendLine("请告诉我是否确认执行，或需要调整哪些步骤。");
    onAiMessage.Invoke(sb.ToString());
}

return plan;
```

## 问题二：步骤执行过程中 AI 完全静默，用户无任何过程反馈

> **状态：✅ 已修复**

### 2.1 现象

执行阶段 Feed 只显示开始和结束两行，子智能体推理过程完全不可见。

### 2.2 根本原因

`ExecuteStepAsync` 方法签名没有 `onAiMessage` 参数，子智能体输出只写日志不推 UI。

### 2.3 实际修复

1. `IOrchestratorAgent.ExecuteStepAsync` 签名增加 `Action<string>? onAiMessage` 参数
2. `TaskExecutionEngine.ExecuteStepWithRetryAsync` 通过 `CreateAiMessageCallback(taskId, stepId: step.StepId)` 传入带 StepId 的回调
3. AI 消息通过 `Events.OnWorkflowConversationMessage` 发布，`WorkflowConversationMessageArgs` 携带 `StepId`
4. `P4TaskDetailVm` 收到后路由到对应的**步骤执行卡片**（`Role="step_card"`）中累积显示

## 问题五：执行阶段 UI 为纯文本行，缺少时间线布局和步骤子过程详情

> **状态：✅ 已修复**

### 5.1 现象

执行阶段所有步骤事件全部渲染为 11px 小字事件行，缺少时间线圆点和步骤子过程详情。

### 5.2 实际修复（整体架构重写）

采用"竖线时间线 + 折叠式步骤卡片"方案，完全替换原有纯文本行设计。
实际实现与文档 08 §2.3 最初描述（`└` 缩进行）不同，改用更现代的折叠卡片。

**5.2.1 数据模型层（`ConversationMessageVm.cs`）**

保留统一 `Role` 字段，新增 `ProgressKind` 枚举和 `Role="step_card"` 类型：

- `Role="progress"` + `ProgressKind` 枚举（PhaseStart / PhaseEnd / StepStart / StepEnd /
  StepFail / StepAux / Detail / Plan / Validation / Lifecycle）
- `Role="step_card"` — 折叠式步骤执行卡片，AI 输出在内部累积
- 时间线属性：`DotColor`、`TimelineTitle`、`TimelineSubText`、`IsLastProgress`
- 卡片属性：`StepId`、`CardTitle`、`IsExpanded`、`IsCardCompleted`、`ContentUpdated` 事件

**5.2.2 ViewModel 层（`P4TaskDetailVm.cs`）**

核心变更：

- `AppendTimeline(kind, title, dotColor, subText)` — 追加时间线节点，自动维护 `IsLastProgress`
- `_activeStepId` / `_activeStepCard` / `_activePhaseCard` — 跟踪当前活跃步骤/阶段卡片
- `OnTaskStepStarted`：追加 `StepStart` 时间线节点 + 创建 `step_card` 消息
- `OnWorkflowConversationMessage`：带 `StepId` 的 AI 消息通过 `TryRouteToStepCard()` 路由到卡片内部累积
- `OnTaskStepCompleted`：调用 `CompleteStepCard()` 折叠卡片 + 追加 `StepEnd` 时间线节点

**5.2.3 View 层（`WorkflowDetailView.axaml`）**

`ItemTemplate` 内两个主要渲染分支：

1. **时间线进度节点**（`IsVisible="{Binding IsProgress}"`）：
   - Grid 两列布局：左列 24px 竖线 + 圆点，右列文本
   - 竖线 1.5px `#333333` 贯穿（非最后一条时全高，最后一条只画上半段）
   - 圆点按 `ProgressKind` 分为阶段级（10px）/ 步骤级（8px）/ 其他（8px）
   - 文本按类型切换：13px 标题行 / 11px 灰色详情行
   - `TimelineSubText` 作为副文本（步骤完成时的结果摘要）

2. **步骤执行卡片**（`IsVisible="{Binding IsStepCard}"`）：
   - Grid 两列：左列虚线延续时间线，右列暗色折叠卡片
   - 卡片标题栏可点击折叠/展开（`OnStepCardHeaderClick`）
   - 内部含 `MarkdownRenderer` + `ScrollViewer`（MaxHeight=320）
   - 流式光标 `▌`（暗绿 `#4A7A5A`）
   - 步骤完成后自动折叠、显示 ✓ 标记

**5.2.4 与文档 08 §2.3 的差异说明**

- 原设计：`└` 缩进的子过程纯文本行，步骤内容平铺
- 实际实现：折叠式卡片 + MarkdownRenderer，子过程内容在卡片内滚动展示
- 优势：步骤输出通常很长（数百行），折叠卡片避免撑爆时间线，完成后自动折叠保持紧凑

## 问题六：对话模式消息缺少左右边距，窗口缩小时右侧文字被压缩

> **状态：✅ 已修复**

### 6.1 现象

用户消息气泡和 AI 消息区域均撑满 ScrollViewer 可用宽度，视觉上贴着右边界。

### 6.2 实际修复

修改文件：`Src/Netor.Cortana.UI/Views/Workspace/WorkflowDetailView.axaml`

`ItemsControl` 整体 `Margin="16,10,16,60"`，内部各角色模板均带 `Margin="8,N,8,N"`：

- 用户气泡 `Border`：`Margin="8,4,8,4"`
- AI 消息 `StackPanel`：`Margin="8,4,8,4"`
- 时间线进度节点 `Grid`：`Margin="8,0,8,0"`
- 步骤卡片 `Grid`：`Margin="8,0,8,0"`
- 系统消息 `TextBlock`：`Margin="8,2,8,2"`
- 工具授权卡片 `Border`：`Margin="8,4,8,4"`

效果：ItemsControl 16px + 自身 8px = 两侧各 **24px** 总边距，内容不再贴边。

   改走 `_runner.RunWithToolsAsync`，否则保持原有 `RunMultiTurnAsync` 逻辑
