# 执行计划(对话流内实时过程卡片) : 100%

## 背景

当前 AI 对话界面只显示文本气泡。AI 在执行 PowerShell 命令、调用工具、思考时，用户看不到任何过程信息，体验上"卡住一片黑"。

同时调研发现：

- 历史回放显示工具过程没有意义，用户不会回头看。
- 数据库中的工具链记录 (`ContentsJson`) 仅用于 AI 协议恢复，不应展示给用户。
- 在 **当前这一轮对话运行期间** 显示折叠工具卡片，能显著提升临场体验。

因此本计划只解决"实时过程显示"这一件事，不动历史 UI、不动数据库展示逻辑。

---

## 目标

在当前对话流中插入一种**实时过程卡片**，用于显示：

- 工具调用过程 (function call / tool call / MCP call)
- 命令执行过程 (PowerShell stdout / stderr)
- 思维过程 (reasoning content)

卡片只服务于"正在运行的这一轮"，不进入历史回放。

---

## 显示效果

### 运行中

```text
用户：帮我检查项目编译

▾ PowerShell · 运行中
  dotnet build
  Determining projects to restore...
  Build succeeded.

小娜：我正在检查项目编译状态...
```

### 接收完成后立即折叠
按照当前计划文档进行修改

```text
▸ PowerShell · 成功 · 12.4s · 退出码 0
```

### 用户手动展开

左侧箭头 `▸ / ▾` 切换，无文字提示。

---

## 设计原则

1. 只显示实时过程，不做历史回放。
2. 实时事件只发给 UI，不污染 WebSocket / 语音通道。
3. 卡片接收完成立即自动折叠，不等待时间。
4. 用户手动点击箭头展开/折叠后，尊重用户意图，不再被任何机制自动覆盖。
5. 高频输出做 UI 节流，避免渲染卡顿。
6. 卡片是消息流里的临时项，不写入数据库展示路径。

### 2. 新增实时过程事件模型

```csharp
public sealed class RealtimeProcessEvent
{
    public string TurnId { get; init; } = "";
    public string ProcessId { get; init; } = "";
    public string Kind { get; init; } = "";      // tool / command / thinking / agent
    public string Title { get; init; } = "";
    public string Status { get; init; } = "";    // running / success / failed / cancelled
    public string Content { get; init; } = "";   // 详细内容，运行中可阅读
    public int? ExitCode { get; init; }
    public long DurationMs { get; init; }
    public DateTimeOffset Timestamp { get; init; }
}
```

`Content` 是详细内容，不是摘要。运行中允许阅读；接收完成后由 UI 自动折叠隐藏，需要时手动展开查看。

### 3. 新增实时过程卡片控件

位置建议：

- 控件：`Src/Netor.Madorin.UI/Controls/RealtimeProcessCard.axaml(.cs)`
- 入口方法：在 `Src/Netor.Madorin.UI/Views/Main/MainWindow.Messaging.cs` 新增

```csharp
RealtimeProcessCardHandle AddRealtimeProcessCard(RealtimeProcessEvent initial);
```

返回 handle，外部通过 handle 持续更新同一张卡片：

```csharp
public sealed class RealtimeProcessCardHandle
{
    public string ProcessId { get; }
    public void UpdateStatus(string status, int? exitCode, long durationMs);
    public void AppendContent(string content);
    public void Complete(string status, int? exitCode, long durationMs);
}
```

卡片视觉结构：

```text
Border
  StackPanel
    Header (Button)
      ▸ / ▾ 箭头 + 图标 + 标题 + 状态 + 耗时
    DetailPanel (默认显示)
      内容区 (节流刷新的 TextBlock / MarkdownRenderer)
```

样式参考已有的 `AddSystemNotice(...)` 折叠样式。

### 4. 自动折叠机制

唯一规则：

- 运行中：始终展开。
- 收到 `Complete` 后：立即折叠（不等待）。
- 用户手动点击箭头：标记 `ManualToggled = true`，从此自动折叠不再生效。

### 5. UI 节流刷新

卡片内部维护：

```csharp
StringBuilder _contentBuffer;
DispatcherTimer _flushTimer;       // 100~200ms
bool _isDirty;
```

`AppendContent` 只追加到 buffer 并标记 dirty。  
`_flushTimer` Tick 时若 dirty 则一次性刷新 UI 文本，避免每行触发渲染。

`Complete` 调用时立即 flush 一次，再立即折叠。

### 6. 在 UI 通道里实现 `IRealtimeProcessOutput`

文件：`Src/Netor.Madorin.UI/Channels/UiChatOutputChannel.cs`

让 `UiChatOutputChannel` 同时实现新接口，内部维护：

```csharp
Dictionary<string, RealtimeProcessCardHandle> _cardsByProcessId;
```

事件处理：

```text
status = running 且 cards 中没有 ProcessId
  -> AddRealtimeProcessCard 创建卡片

status = running 且已有卡片
  -> AppendContent

status = success / failed / cancelled
  -> Complete (内部立即 flush + 立即折叠)
  -> 从字典移除
```

### 7. 事件源接入

#### 7.1 PowerShell 命令实时输出

文件：`Src/Netor.Madorin.Plugin/BuiltIn/PowerShell/PowerShellProvider.cs`

依赖注入 `IRealtimeProcessOutput`（如果未注册则跳过）。

`ExecutePowerShellAsync` 中：

- 进入方法前：发送 `running` + `Title = "PowerShell"` + `Content = script`
- 订阅 `_executor.OnOutputLineReceived` → `running` 状态追加 `Content`
- 订阅 `_executor.OnErrorReceived` → `running` 状态追加 `Content`（错误流可加 `[stderr]` 前缀）
- 正常返回：`success` + 退出码 + 耗时
- 异常 / 超时 / 取消：`failed` / `cancelled` + 耗时

同样适用于 `start_session` / `send_command` 等长任务方法。

#### 7.2 通用工具调用

文件：`Src/Netor.Madorin.AI/AiChatHostedService.cs`

在 `RunStreamingAsync` 的循环里：

- 解析 `chunk.Contents`，识别：
  - `FunctionCallContent` / `McpServerToolCallContent` / `ToolCallContent` → 发 `running`
  - `FunctionResultContent` / `McpServerToolResultContent` / `ToolResultContent` → 发 `success` / `failed`
- 仅通过 `IRealtimeProcessOutput` 发送，不走 `IAiOutputChannel`。

注意：PowerShell 工具已经在 7.1 自己发了卡片，要避免重复创建。  
做法：以 `ProcessId = CallId` 为主键，PowerShell 主动创建后通用层探测到同 CallId 就跳过。

#### 7.3 思维过程

文件：`Src/Netor.Madorin.AI/AiChatHostedService.cs`

`chunk.Contents` 中遇到 `TextReasoningContent`：

- `ProcessId = "reasoning-" + turnId`
- `Kind = "thinking"`
- `Status = running`
- `Content = reasoning.Text`

当本轮 streaming 结束且没有更多 reasoning chunk 时，发送 `success` 关闭卡片（立即折叠）。

## 验收

- 让 AI 执行一条 `dotnet build`，能看到 PowerShell 卡片实时滚动输出。
- 执行结束后卡片立即折叠为一行：工具名、状态、退出码、耗时。
- 点击左侧箭头能再次展开查看完整输出。
- 手动展开后，不会被任何机制自动折叠。
- 历史会话切换后，过往工具卡片不再出现（历史 UI 保持现状）。
- WebSocket 客户端和语音通道不会收到实时过程事件。
- 输出量大时（如 `dir /s`）界面不卡顿。
