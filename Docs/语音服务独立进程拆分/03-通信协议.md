# 03 - 通信协议

## 概览

| 服务 | 角色 | 地址 | 端口 |
|---|---|---|---|
| STT 语音识别 | WebSocket Server | `ws://localhost:12858/` | 12858 |
| TTS 语音合成 | WebSocket Server | `ws://localhost:12859/` | 12859 |

- 协议：标准 WebSocket (RFC 6455)
- 编码：UTF-8 JSON 文本帧
- 主程序是 **客户端**，连接到 STT/TTS 服务
- 心跳：服务端每 30 秒发 `{"type":"heartbeat"}`

## 通用消息格式

```json
{
  "type": "消息类型",
  "data": "载荷（可选）"
}
```

所有消息都是 JSON 对象，必须包含 `type` 字段，`data` 字段可选。

## STT 服务协议 (`ws://localhost:12858/`)

### 连接建立

服务端在 WebSocket 握手成功后立即推送：

```json
{
  "type": "connected",
  "data": "{\"model\":\"paraformer-streaming\",\"version\":\"1.0.0\"}"
}
```

### 主程序 → STT（命令）

| type | data | 说明 |
|---|---|---|
| `start` | — | 开始一次识别会话，服务端开始录音+解码 |
| `cancel` | — | 取消当前识别，停止录音，不再发 `final` |
| `config` | JSON | 运行时参数更新（见下方） |

#### `config` 载荷

```json
{
  "type": "config",
  "data": "{\"rule1MinTrailingSilence\":2.4,\"rule2MinTrailingSilence\":1.2,\"rule3MinUtteranceLength\":20.0,\"recognitionTimeoutSeconds\":10.0}"
}
```

服务端收到后立即生效，影响下一次 `start`。

### STT → 主程序（事件）

| type | data | 说明 |
|---|---|---|
| `connected` | 模型/版本 JSON | 连接建立 |
| `partial` | 中间文本 | 识别中间结果（实时更新） |
| `final` | 最终文本 | 一句话识别完成 |
| `stopped` | — | 识别会话结束（超时/无内容/cancel 后的确认） |
| `error` | 错误描述 | 异常信息 |
| `heartbeat` | — | 心跳 |

### STT 状态机

```
[空闲] ──start──→ [录音中] ──识别到文本──→ 发 partial (持续)
                    │                          │
                    │                     自然停顿/超时
                    │                          │
                    │                          ▼
                    │                    发 final + stopped
                    │
                    ├──cancel──→ 发 stopped
                    └──超时无内容──→ 发 stopped
```

## TTS 服务协议 (`ws://localhost:12859/`)

### 连接建立

```json
{
  "type": "connected",
  "data": "{\"model\":\"melo-tts-zh_en\",\"version\":\"1.0.0\",\"sampleRate\":44100}"
}
```

### 主程序 → TTS（命令）

| type | data | 说明 |
|---|---|---|
| `speak` | 完整文本 | 合成并播放全部文本（阻塞式，完成发 `completed`） |
| `enqueue` | 一句话文本 | 流式入队，配合 `finish` 使用 |
| `finish` | — | 标记文本输入完毕，合成完剩余队列后发 `completed` |
| `stop` | — | 立即停止合成和播放，不发 `completed` |
| `greeting_play` | — | **隐式停止**当前播放 + 播放预缓存问候语 |
| `greeting_regen` | 新文本（可选） | 重新生成问候语缓存，空则用当前配置 |
| `config` | JSON | 运行时参数更新（见下方） |

#### `config` 载荷

```json
{
  "type": "config",
  "data": "{\"speed\":1.0,\"welcomeGreeting\":\"主人，我在!\"}"
}
```

#### `greeting_play` 行为说明

1. 如果当前有正在进行的合成/播放，**先停止**
2. 播放预缓存的问候语音频
3. 播放完成后发 `completed`
4. 如果缓存不存在，实时合成后播放

#### 流式入队典型序列

```
主程序 → TTS: {"type":"enqueue","data":"欢迎使用智能语音助手。"}
主程序 → TTS: {"type":"enqueue","data":"今天天气不错，"}
主程序 → TTS: {"type":"enqueue","data":"适合出去走走。"}
主程序 → TTS: {"type":"finish"}

TTS → 主程序: {"type":"started"}
TTS → 主程序: {"type":"subtitle","data":"欢迎使用智能语音助手。"}
TTS → 主程序: {"type":"subtitle","data":"今天天气不错，"}
TTS → 主程序: {"type":"subtitle","data":"适合出去走走。"}
TTS → 主程序: {"type":"completed"}
```

### TTS → 主程序（事件）

| type | data | 说明 |
|---|---|---|
| `connected` | 模型/版本 JSON | 连接建立 |
| `started` | — | 开始播放（流水线启动） |
| `subtitle` | 当前句子文本 | 正在播放的句子，用于主程序气泡显示 |
| `completed` | — | 全部播放完成 |
| `error` | 错误描述 | 异常信息 |
| `heartbeat` | — | 心跳 |

### TTS 状态机

```
[空闲] ──enqueue──→ [入队中] ──finish──→ [播放中] ──播完──→ 发 completed → [空闲]
  │                    │                    │
  │                    │                    ├── 发 subtitle（每句）
  │                    │                    │
  ├──speak──→ [播放中] ──播完──→ 发 completed → [空闲]
  │                    │
  ├──greeting_play──→ [停止旧的] → [播放问候] → 发 completed → [空闲]
  │
  └── 任何状态 ──stop──→ 立即停止 → [空闲]（不发 completed）
```

## AOT 安全要求

协议层的 JSON 序列化必须使用 Source Generator：

```csharp
// ✅ AOT 安全
[JsonSerializable(typeof(VoiceServiceMessage))]
[JsonSourceGenerationOptions(DefaultIgnoreCondition = JsonIgnoreCondition.WhenWritingNull)]
internal partial class VoiceProtocolJsonContext : JsonSerializerContext;
```

服务端和客户端共享 `Netor.Cortana.Voice.Protocol` 类库中的类型定义。
