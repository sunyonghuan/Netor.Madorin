# 01 - 架构设计

## 目标架构总览

```
┌──────────────────────────────────────────┐
│ 主程序 (Cortana)                          │
│                                          │
│ WakeWordService    ← 留在主进程           │
│ VoiceOrchestrator  ← 新增：协调整条链路   │
│ SttProxyService    ← 新增：WS 客户端代理  │
│ TtsProxyService    ← 新增：WS 客户端代理  │
│ VoiceProcessManager← 新增：子进程生命周期  │
│                                          │
│ EventHub 事件体系不变                     │
│ IAiInputChannel / IAiOutputChannel 不变   │
│ IVoiceCoordinator 接口不变               │
└──────────┬────────────────┬──────────────┘
           │ WS :12858      │ WS :12859
           ▼                ▼
┌──────────────────┐ ┌──────────────────┐
│ Cortana.Stt.exe  │ │ Cortana.Tts.exe  │
│                  │ │                  │
│ 麦克风录制        │ │ 文字→音频合成     │
│ 语音识别引擎      │ │ 音频播放          │
│ 模型(STT/)       │ │ 模型(TTS/)       │
│ 独立更新/替换     │ │ 独立更新/替换     │
└──────────────────┘ └──────────────────┘
```

## 进程职责划分

### 主进程 (Cortana)

| 组件 | 职责 |
|---|---|
| `WakeWordService` | 麦克风监听 + 唤醒词检测（现有，不变） |
| `VoiceOrchestrator` | **新增**。接管原 `SpeechRecognitionService` 中的编排逻辑：唤醒→问候→识别→AI→合成→续杯 |
| `SttProxyService` | **新增**。WS 客户端连接 STT 进程，收发消息，转发事件到 EventHub |
| `TtsProxyService` | **新增**。WS 客户端连接 TTS 进程，收发消息，转发事件到 EventHub |
| `SttProxyInputChannel` | **新增**。实现 `IAiInputChannel`，替代现有 `VoiceInputChannel` |
| `TtsProxyOutputChannel` | **新增**。实现 `IAiOutputChannel`，替代现有 `VoiceChatOutputChannel` |
| `VoiceCoordinator` | **改造**。`CancelEverything()` 通过 Proxy 发 WS cancel/stop |
| `VoiceProcessManager` | **新增**。管理 Stt.exe/Tts.exe 的启动/监控/重启/关闭 |

### STT 独立进程 (Cortana.Stt.exe)

| 组件 | 职责 |
|---|---|
| WebSocket Server | 监听 `:12858`，接收主程序命令 |
| 麦克风录制 | `AotWaveRecorder` + winmm P/Invoke |
| 识别引擎 | Sherpa-ONNX Paraformer Streaming |
| 模型管理 | 从 `sherpa_models/STT/` 加载，支持命令行指定路径 |

**不包含**：AI 引擎、TTS 引擎、UI 控制、EventHub

### TTS 独立进程 (Cortana.Tts.exe)

| 组件 | 职责 |
|---|---|
| WebSocket Server | 监听 `:12859`，接收主程序命令 |
| 合成引擎 | Sherpa-ONNX MeloTTS VITS（或未来其他模型） |
| 双 Channel 流水线 | 文本队列→合成线程→音频队列→播放线程 |
| 音频播放 | `AotWavePlayer` + `SegmentedWaveProvider` |
| 问候语缓存 | 预生成并缓存欢迎语音频 |
| 断句 + 文本清洗 | Markdown 清理、中英文断句 |
| 模型管理 | 从 `sherpa_models/TTS/` 加载 |

**不包含**：AI 引擎、STT 引擎、UI 控制、EventHub

## 编排流程（VoiceOrchestrator 状态机）

```
唤醒词触发 (WakeWordService → EventHub: OnWakeWordDetected)
  │
  ├─① VoiceOrchestrator 收到事件
  ├─② 发 WS 给 STT: cancel（取消旧识别）
  ├─③ 发 WS 给 TTS: greeting_play（隐式停止旧播放 + 播问候语）
  ├─④ 等待 TTS: completed
  ├─⑤ 发 WS 给 STT: start（启动识别）
  ├─⑥ 收到 STT: partial → EventHub: OnSttPartial → UI 气泡 + WS 广播
  ├─⑦ 收到 STT: final → chatEngine.SendMessageAsync（AI 推理）
  ├─⑧ AI token → TtsProxyOutputChannel → WS 给 TTS: enqueue
  ├─⑨ AI done → WS 给 TTS: finish
  ├─⑩ 收到 TTS: subtitle → EventHub: OnTtsSubtitle → UI 气泡
  ├─⑪ 收到 TTS: completed → EventHub: OnTtsCompleted
  ├─⑫ 判断 IsMainWindowVisible？
  │    ├→ 可见：发 EventHub: OnChatCompleted，结束本轮
  │    └→ 不可见：回到 ⑤ 续杯
  └─ 循环直到超时或新唤醒打断
```

### 用户主动取消（点击停止按钮/发送消息）

```
MainWindow 调 IVoiceCoordinator.CancelEverything()
  │
  ├─ VoiceCoordinator → SttProxy: cancel
  ├─ VoiceCoordinator → TtsProxy: stop
  ├─ VoiceCoordinator → chatEngine.CancelCurrentTask()
  └─ VoiceOrchestrator 重置状态
```
