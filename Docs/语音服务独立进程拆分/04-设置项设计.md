# 04 - 设置项设计

## 新增设置项

在 `SystemSettingsService.SeedDefaults()` 中新增以下设置项，
添加到现有"语音唤醒"分组之前，作为独立的"语音服务"分组。

### 设置项定义

| Key | 分组 | 显示名 | 类型 | 默认值 | 说明 |
|---|---|---|---|---|---|
| `Voice.WakeWord.Enabled` | 语音服务 | 启用语音唤醒 | bool | true | 控制 KWS 服务是否启动 |
| `Voice.Stt.Enabled` | 语音服务 | 启用语音识别 | bool | true | 控制 STT 子进程是否拉起 |
| `Voice.Tts.Enabled` | 语音服务 | 启用语音合成 | bool | true | 控制 TTS 子进程是否拉起 |

### Seed 代码示例

```csharp
// ── 语音服务开关 ────────────────────
Seed("Voice.WakeWord.Enabled",
    group: "语音服务", displayName: "启用语音唤醒",
    description: "关闭后不再监听唤醒词，避免误触发。需要重启应用生效。",
    value: "true", valueType: "bool", sortOrder: 0),

Seed("Voice.Stt.Enabled",
    group: "语音服务", displayName: "启用语音识别",
    description: "关闭后不启动语音识别服务进程，节省系统资源。需要重启应用生效。",
    value: "true", valueType: "bool", sortOrder: 1),

Seed("Voice.Tts.Enabled",
    group: "语音服务", displayName: "启用语音合成",
    description: "关闭后不启动语音合成服务进程，AI 回复仅以文字显示。需要重启应用生效。",
    value: "true", valueType: "bool", sortOrder: 2),
```

## 开关生效逻辑

### WakeWordService（主进程内）

```csharp
public Task StartAsync(CancellationToken cancellationToken)
{
    // 检查开关
    if (!systemSettings.GetValue("Voice.WakeWord.Enabled", true))
    {
        logger.LogInformation("语音唤醒已禁用，跳过启动");
        return Task.CompletedTask;
    }
    // ... 原有启动逻辑
}
```

### VoiceProcessManager（管理子进程）

```csharp
public Task StartAsync(CancellationToken cancellationToken)
{
    if (systemSettings.GetValue("Voice.Stt.Enabled", true))
    {
        StartSttProcess();
    }
    else
    {
        logger.LogInformation("语音识别服务已禁用，不启动 Cortana.Stt.exe");
    }

    if (systemSettings.GetValue("Voice.Tts.Enabled", true))
    {
        StartTtsProcess();
    }
    else
    {
        logger.LogInformation("语音合成服务已禁用，不启动 Cortana.Tts.exe");
    }

    return Task.CompletedTask;
}
```

### VoiceOrchestrator 编排器

```csharp
// 唤醒触发时检查各服务可用性
private async Task OnWakeWordDetected()
{
    // TTS 未启用 → 跳过问候，直接进 STT
    if (TtsProxy.IsConnected)
    {
        await TtsProxy.SendAsync("greeting_play");
        await WaitForTtsCompleted();
    }

    // STT 未启用 → 无法识别，结束本轮
    if (!SttProxy.IsConnected)
    {
        logger.LogWarning("STT 服务未连接，无法启动语音识别");
        return;
    }

    await SttProxy.SendAsync("start");
    // ...
}
```

### TtsProxyOutputChannel（IAiOutputChannel）

```csharp
public bool IsActive
{
    get
    {
        // 主窗口可见时不走 TTS（前端已通过 WebSocket 接收）
        if (windowController.IsMainWindowVisible()) return false;
        // TTS 服务未连接时不活跃
        if (!ttsProxy.IsConnected) return false;
        return true;
    }
}
```

## 联动关系

| 场景 | WakeWord | STT | TTS | 行为 |
|---|---|---|---|---|
| 全部开启 | ✅ | ✅ | ✅ | 完整语音对话 |
| 关闭唤醒 | ❌ | ✅ | ✅ | 无法语音触发，但 STT/TTS 服务仍运行（未来可手动触发） |
| 关闭 STT | ✅ | ❌ | ✅ | 唤醒后只能播问候语，无法识别用户语音 |
| 关闭 TTS | ✅ | ✅ | ❌ | 唤醒后可识别语音并 AI 回复，但只显示文字不播报 |
| 全部关闭 | ❌ | ❌ | ❌ | 纯文字模式，无录音/播放，节省最多资源 |

## UI 显示

设置页面"语音服务"分组显示在最前面，三个 ToggleSwitch 开关一目了然：

```
┌─────────────────────────────────────────┐
│ 语音服务                                 │
├─────────────────────────────────────────┤
│ 启用语音唤醒        [开/关]              │
│ 关闭后不再监听唤醒词，避免误触发          │
├─────────────────────────────────────────┤
│ 启用语音识别        [开/关]              │
│ 关闭后不启动语音识别服务进程              │
├─────────────────────────────────────────┤
│ 启用语音合成        [开/关]              │
│ 关闭后AI回复仅以文字显示                  │
└─────────────────────────────────────────┘
```

现有的"语音唤醒"、"语音识别"、"语音合成"三个分组仍然保留，
用于调节具体参数（灵敏度、超时等），与服务开关互不影响。
