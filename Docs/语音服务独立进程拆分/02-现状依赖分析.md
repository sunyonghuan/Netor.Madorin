# 02 - 现状依赖分析

## Voice 模块文件清单

> 项目：`Src/Netor.Cortana.Voice/`
> 依赖 NuGet：`org.k2fsa.sherpa.onnx` 1.12.35、`NAudio.WinMM` 2.3.0、`Netor.EventHub`
> 依赖项目：`Netor.Cortana.Entitys`

| 文件 | 职责 | 拆分去向 |
|---|---|---|
| `WakeWordService.cs` | KeywordSpotter 唤醒词检测 | **留主进程** |
| `SpeechRecognitionService.cs` | Paraformer Streaming 流式识别 + **编排逻辑** | 引擎→STT 进程，编排→主进程 |
| `TextToSpeechService.cs` | MeloTTS VITS 合成 + 双 Channel 流水线 + 播放 | **TTS 进程** |
| `VoiceChatOutputChannel.cs` | AI token → 断句 → TTS enqueue 桥接 | **替换**为 `TtsProxyOutputChannel` |
| `VoiceInputChannel.cs` | STT final → chatEngine.SendMessageAsync 桥接 | **替换**为 `SttProxyInputChannel` |
| `IVoiceCoordinator.cs` | 取消所有语音/AI 任务统一入口 | 接口不变，**实现改造** |
| `AotWavePlayer.cs` | winmm waveOut P/Invoke 播放 | **TTS 进程** |
| `AotWaveRecorder.cs` | winmm waveIn P/Invoke 录制 | **STT 进程 + 主进程各一份** |
| `SegmentedWaveProvider.cs` | 分段音频提供器（字幕同步） | **TTS 进程** |
| `VoiceServiceExtensions.cs` | DI 注册扩展方法 | **改造**（切换到 Proxy 组件） |
| `VoiceChatService.cs` | 空壳类（未使用） | **删除** |

## 详细依赖关系图

### SpeechRecognitionService（耦合最重，拆分核心）

```
SpeechRecognitionService
  ├── 构造注入
  │   ├── ILogger<SpeechRecognitionService>
  │   ├── IAppPaths                    ← 模型路径
  │   ├── SystemSettingsService        ← 读取 STT 参数
  │   ├── IPublisher                   ← 发布事件
  │   ├── ISubscriber                  ← 订阅事件
  │   ├── TextToSpeechService          ← ⚠️ 直接调 PlayGreetingAsync/Stop
  │   ├── IAiChatEngine                ← ⚠️ 直接调 CancelCurrentTask
  │   └── IWindowController            ← ⚠️ 判断 IsMainWindowVisible
  │
  ├── 订阅事件
  │   ├── Events.OnWakeWordDetected    ← 触发：取消旧轮次 + 播问候 + 启动STT
  │   └── Events.OnChatCompleted       ← 触发：续杯（条件：窗口不可见 + 轮次未取消）
  │
  ├── 发布事件
  │   ├── Events.OnSttPartial          ← 识别中间结果
  │   ├── Events.OnSttFinal            ← 识别最终结果
  │   └── Events.OnSttStopped          ← 识别会话结束
  │
  └── 读取配置
      ├── SherpaOnnx.Rule1MinTrailingSilence
      ├── SherpaOnnx.Rule2MinTrailingSilence
      ├── SherpaOnnx.Rule3MinUtteranceLength
      └── SherpaOnnx.RecognitionTimeoutSeconds
```

**⚠️ 标记的 3 个依赖是拆分的核心障碍**，它们使 STT 引擎承担了编排职责。

### TextToSpeechService

```
TextToSpeechService
  ├── 构造注入
  │   ├── ILogger<TextToSpeechService>
  │   ├── IAppPaths                    ← 模型路径
  │   ├── SystemSettingsService        ← 读取 Tts.Speed / Tts.WelcomeGreeting
  │   ├── IPublisher                   ← 发布事件
  │   └── ISubscriber                  ← 订阅事件
  │
  ├── 订阅事件
  │   ├── Events.OnTtsEnqueue          ← 收到句子文本，入队合成
  │   └── Events.OnTtsFinish           ← AI 推理完成，没有后续文本了
  │
  ├── 发布事件
  │   ├── Events.OnTtsStarted          ← 流水线启动
  │   ├── Events.OnTtsSubtitle         ← 当前播放的句子（字幕）
  │   ├── Events.OnTtsCompleted        ← 全部播放完成
  │   └── Events.OnChatCompleted       ← 完整对话流程结束
  │
  └── 读取配置
      ├── Tts.Speed
      └── Tts.WelcomeGreeting
```

**无 ⚠️ 标记**：TTS 自身耦合度低，通过事件驱动，拆分相对安全。

### WakeWordService

```
WakeWordService
  ├── 构造注入
  │   ├── ILogger<WakeWordService>
  │   ├── IAppPaths                    ← 模型路径
  │   ├── SystemSettingsService        ← 读取 KWS 参数
  │   └── IPublisher                   ← 发布事件
  │
  ├── 发布事件
  │   └── Events.OnWakeWordDetected    ← 检测到唤醒词
  │
  └── 读取配置
      ├── SherpaOnnx.KeywordsThreshold
      ├── SherpaOnnx.KeywordsScore
      └── SherpaOnnx.NumTrailingBlanks
```

**无外部依赖**，完全自包含，留在主进程零风险。

### VoiceChatOutputChannel

```
VoiceChatOutputChannel : IAiOutputChannel
  ├── 构造注入
  │   ├── ILogger
  │   ├── IPublisher                   ← 发布 OnTtsEnqueue / OnTtsFinish
  │   └── IWindowController            ← IsActive = !IsMainWindowVisible()
  │
  └── 实现方法
      ├── OnTokenAsync()               ← AI token → 断句 → OnTtsEnqueue
      ├── OnDoneAsync()                ← flush + OnTtsFinish
      ├── OnCancelledAsync()           ← clear + OnTtsFinish
      └── OnErrorAsync()               ← clear + OnTtsFinish
```

**拆分后替换为 `TtsProxyOutputChannel`**：
- `OnTokenAsync` → 断句 → WS `enqueue`
- `OnDoneAsync` → WS `finish`
- `OnCancelledAsync` → WS `stop`

### VoiceInputChannel

```
VoiceInputChannel : IAiInputChannel, IHostedService
  ├── 构造注入
  │   ├── ILogger
  │   ├── IAiChatEngine                ← ⚠️ 直接调 SendMessageAsync
  │   ├── IPublisher
  │   └── ISubscriber                  ← 订阅 Events.OnSttFinal
  │
  └── 行为
      └── 收到 OnSttFinal → chatEngine.SendMessageAsync()
```

**拆分后替换为 `SttProxyInputChannel`**：收到 WS `final` 事件 → 调 `chatEngine.SendMessageAsync`。

### VoiceCoordinator

```
VoiceCoordinator : IVoiceCoordinator
  ├── 构造注入
  │   ├── SpeechRecognitionService     ← ⚠️ 直接调 CancelCurrentRound()
  │   └── IPublisher                   ← 发布 OnSttStopped
  │
  └── 方法
      └── CancelEverything()
          ├─ stt.CancelCurrentRound()  ← 内部取消 AI + TTS
          └─ publish OnSttStopped
```

**拆分后改造**：`CancelEverything()` → SttProxy.cancel + TtsProxy.stop + chatEngine.CancelCurrentTask。

## Networks 层消费 Voice 事件

`WebSocketEventRelayService.cs` 订阅以下事件并广播到前端 WebSocket：

| EventHub 事件 | WS 广播类型 |
|---|---|
| `OnSttPartial` | `stt_partial` |
| `OnSttFinal` | `stt_final` |
| `OnSttStopped` | `stt_stopped` |
| `OnTtsStarted` | `tts_started` |
| `OnTtsSubtitle` | `tts_subtitle` |
| `OnTtsCompleted` | `tts_completed` |
| `OnChatCompleted` | `chat_completed` |
| `OnWakeWordDetected` | `wakeword_detected` |

**拆分后这些全部不变**——Proxy 收到独立进程的 WS 消息后转发到 EventHub，
Networks 层继续订阅 EventHub 事件广播。对前端 WebSocket 客户端完全透明。

## Entitys 层定义的语音相关接口和事件

### 接口（均在 `Netor.Cortana.Entitys/Interfaces/`）

| 接口 | 实现者 | 拆分影响 |
|---|---|---|
| `IAiOutputChannel` | `VoiceChatOutputChannel` → `TtsProxyOutputChannel` | 接口不变，实现替换 |
| `IAiInputChannel` | `VoiceInputChannel` → `SttProxyInputChannel` | 接口不变，实现替换 |
| `IAiChatEngine` | `AiChatHostedService`（AI 层） | 不变 |
| `IWindowController` | AvaloniaUI 层 | 不变 |
| `IVoiceCoordinator` | `VoiceCoordinator` | 接口不变，实现改造 |

### 事件定义（`Events.cs`）

所有语音事件 ID 和参数类型定义在 Entitys 层，**全部保留不变**：

```
OnWakeWordDetected  → VoiceSignalArgs
OnSttPartial        → VoiceTextArgs
OnSttFinal          → VoiceTextArgs
OnSttStopped        → VoiceSignalArgs
OnTtsStarted        → VoiceSignalArgs
OnTtsSubtitle       → VoiceTextArgs
OnTtsCompleted      → VoiceSignalArgs
OnChatCompleted     → VoiceSignalArgs
OnTtsEnqueue        → TtsEnqueueArgs    ← 拆分后主进程内部可能不再使用
OnTtsFinish         → VoiceSignalArgs    ← 拆分后主进程内部可能不再使用
```

`OnTtsEnqueue` 和 `OnTtsFinish` 原来是 AI→TTS 的进程内事件驱动，
拆分后改为 WS 直接发送 `enqueue`/`finish`，这两个事件可能变为冗余，
但为了兼容性暂时保留。
