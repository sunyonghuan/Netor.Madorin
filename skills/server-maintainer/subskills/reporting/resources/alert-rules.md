# 巡检告警规则配置

## 1. 指标阈值

| 指标 | ⚠️ 警告阈值 | 🔴 严重阈值 | 说明 |
| :--- | :--- | :--- | :--- |
| **CPU 使用率** | > 80% | > 90% | 持续 1 分钟平均值 |
| **内存使用率** | > 70% | > 85% | 已用物理内存占比 |
| **磁盘使用率 (/)** | > 80% | > 90% | 根分区使用率 |
| **磁盘使用率 (/data)** | > 85% | > 95% | 数据分区使用率 |
| **系统负载 (Load)** | > CPU 核数 × 0.8 | > CPU 核数 × 1.2 | 1 分钟负载 |
| **IO 等待 (wa%)** | > 20% | > 40% | 磁盘 IO 瓶颈指标 |
| **Swap 使用率** | > 30% | > 60% | 内存不足指标 |

## 2. 安全告警

| 项目 | ⚠️ 警告阈值 | 🔴 严重阈值 | 说明 |
| :--- | :--- | :--- | :--- |
| **SSH 失败登录** | > 100 次 | > 1000 次 | 过去 24 小时内统计 |
| **僵尸进程** | > 5 个 | > 20 个 | `top` 中的 zombie 数 |
| **运行时间** | - | > 180 天 | 建议定期重启以应用内核更新 |

## 3. 状态判定逻辑

- **✅ 正常**: 所有指标均在警告阈值以下。
- **⚠️ 警告**: 任意一项指标达到警告阈值，但未达到严重阈值。
- **🔴 严重**: 任意一项指标达到严重阈值，或 SSH 失败登录极多。
- **❌ 离线**: 无法通过 SSH 连接获取数据。

## 4. 建议处理策略

- **CPU/内存高**: 使用 `top` 或 `htop` 定位进程，必要时重启服务。
- **磁盘满**: 清理 `/var/log`，检查大文件 `du -sh *`，清理 Docker 镜像。
- **SSH 暴力破解**: 查看 `lastb`，配置 `fail2ban` 或限制 IP 白名单。
- **IO 高**: 使用 `iostat -x 1` 查看具体设备，检查是否有大量日志写入或数据库查询。
